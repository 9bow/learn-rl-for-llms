## 오픈소스 추론 모델의 중요성

DeepSeek-R1의 등장(2025년 1월)은 오픈소스 AI 커뮤니티에 큰 자극이 되었다. 폐쇄적인 상업 모델(GPT-o1, o3 등)에 필적하는 추론 능력을 오픈소스로 재현하려는 경쟁이 시작되었다. 이 섹션은 그 결과로 탄생한 주요 오픈소스 추론 모델들을 살펴본다.

## OpenR1: DeepSeek-R1 완전 재현

**OpenR1** (Hugging Face, 2025)은 DeepSeek-R1의 훈련 파이프라인을 완전히 재현하는 것을 목표로 합니다. 단순히 모델 가중치를 공개하는 것이 아니라, **훈련 방법론 자체** 를 투명하게 공개합니다.

### 재현 전략

DeepSeek-R1의 훈련은 두 핵심 단계로 구성됩니다.

**1단계: 순수 RL (GRPO 기반)**

그룹 상대 정책 최적화(GRPO)를 사용하여 기반 모델을 직접 강화학습으로 훈련합니다. 보상은 수학 문제의 정답 여부와 형식 준수로 구성됩니다.

$$r_{total} = r_{correctness} + r_{format}$$

**2단계: 거부 샘플링 기반 SFT**

1단계에서 훈련된 모델로 대규모 추론 데이터를 생성하고, 고품질 샘플만 선별하여 지도학습에 활용합니다.

### Mixture-of-Thoughts 데이터셋

OpenR1의 핵심 기여 중 하나는 **Mixture-of-Thoughts (MoT)** 데이터셋입니다. 350,000개의 검증된 추론 흔적(reasoning traces)으로 구성되며 다음 특성을 갖습니다.

| 특성 | 세부 내용 |
|------|---------|
| 총 샘플 수 | 350,000 |
| 검증 방법 | 자동 정답 검증 + 인간 검토 |
| 추론 도메인 | 수학, 코드, 과학, 논리 |
| 평균 추론 길이 | 2,847 토큰 |
| 언어 | 영어, 중국어 |

### CodeForces-CoTs 데이터셋

프로그래밍 경시대회 플랫폼인 CodeForces의 문제들에 대한 Chain-of-Thought 데이터셋입니다. 각 문제에 대해 여러 단계의 추론 과정을 포함하며, 코드 생성과 수학적 추론이 결합된 형태다.

## OpenR: 통합 추론 프레임워크

**OpenR** (Wang et al., arXiv:2410.09671)은 단순한 모델이 아닌 **연구 프레임워크** 다. LLM 추론 향상을 위한 데이터 수집, 온라인·오프라인 RL, 비자기회귀 디코딩을 통합합니다.

### 프레임워크 구조

```
데이터 파이프라인
    ├── 자동 문제 생성
    ├── 추론 경로 샘플링
    └── 품질 필터링
         ↓
RL 훈련 엔진
    ├── 온라인 RL (PPO, GRPO)
    ├── 오프라인 RL (DPO, RRHF)
    └── 하이브리드 방식
         ↓
추론 시 최적화
    ├── 빔 서치
    ├── MCTS
    └── 비자기회귀 디코딩
```

### 온라인 vs 오프라인 RL 비교

| 특성 | 온라인 RL | 오프라인 RL |
|------|---------|----------|
| 데이터 생성 | 훈련 중 실시간 | 사전 수집 |
| 탐색 다양성 | 높음 | 낮음 |
| 계산 비용 | 높음 | 낮음 |
| 분포 편향 | 낮음 | 높음 |
| 안정성 | 낮음 | 높음 |
| 적용 사례 | 빠르게 변화하는 도메인 | 안정적 도메인 |

### 비자기회귀 디코딩

전통적인 자기회귀(autoregressive) 디코딩은 토큰을 하나씩 순차적으로 생성합니다. OpenR은 **비자기회귀(non-autoregressive) 디코딩** 을 통해 추론 속도를 높이면서 품질을 유지하는 방법을 탐구합니다. 이는 병렬 토큰 생성과 반복적 정제를 결합합니다.

## Qwen3: 통합 추론 모델

**Qwen3** (Qwen Team, arXiv:2505.09388)는 Alibaba의 대규모 언어 모델 시리즈로, 추론 능력에서 새로운 기준을 세웠다.

### Thinking/Non-Thinking 통합 모드

Qwen3의 가장 독특한 특성은 **단일 모델 내에서 thinking 모드와 non-thinking 모드를 전환** 할 수 있다는 점입니다.

**Thinking 모드** (`/think` 태그): 복잡한 문제에 대해 장문의 내부 추론을 수행합니다. DeepSeek-R1처럼 `<think>...</think>` 블록을 생성한 후 최종 답을 반환합니다.

**Non-thinking 모드** (`/no_think` 태그): 간단한 질문에 대해 빠른 직접 응답을 제공합니다. 불필요한 추론 오버헤드를 줄입니다.

이 설계는 **연산 효율성** 의 측면에서 중요합니다. 모든 질문에 deep thinking이 필요한 것은 아니며, 사용자가 상황에 맞게 선택할 수 있습니다.

### 스케일 범위

| 모델 | 파라미터 | 특징 |
|------|---------|------|
| Qwen3-0.6B | 0.6B | 엣지 디바이스용 |
| Qwen3-1.7B | 1.7B | 경량 추론 |
| Qwen3-4B | 4B | 균형형 |
| Qwen3-8B | 8B | 고성능 소형 |
| Qwen3-14B | 14B | 전문가급 |
| Qwen3-32B | 32B | 고성능 |
| Qwen3-30B-A3B | 30B (MoE, 3B 활성) | 효율적 MoE |
| Qwen3-235B-A22B | 235B (MoE, 22B 활성) | 최고 성능 |

0.6B에서 235B까지의 폭넓은 스케일은 다양한 배포 환경을 지원합니다.

### 훈련 파이프라인

Qwen3의 훈련은 4단계로 구성됩니다.

1. **대규모 사전훈련**: 36조 토큰 이상의 고품질 데이터
2. **장문 추론 SFT**: CoT 데이터로 추론 능력 초기화
3. **순수 RL**: 검증 가능한 보상으로 추론 강화
4. **통합 모드 학습**: thinking/non-thinking 전환 학습

## Qwen2.5-Math: 자기 개선 루프

**Qwen2.5-Math** 는 수학 전문 모델로, **자기 개선(self-improvement) 루프** 를 핵심 훈련 메커니즘으로 활용합니다.

### 자기 개선 메커니즘

```
초기 모델 M_0
    ↓ 문제 풀기
고품질 풀이 선별
    ↓ 지도학습
개선된 모델 M_1
    ↓ 더 어려운 문제 풀기
더 고품질 풀이 선별
    ↓ 지도학습
더 개선된 모델 M_2
    ↓ (반복)
```

이 루프에서 핵심은 **품질 필터** 다. 모델이 생성한 모든 풀이를 사용하는 것이 아니라, 정답이면서 추론 과정이 명확하고 간결한 풀이만을 선별합니다.

$$\mathcal{D}_{t+1} = \{(x, c) | M_t(x) = c \land \text{quality}(c) > \tau\}$$

## 정리

오픈소스 추론 모델 생태계는 2025년 들어 폭발적으로 성장했다. OpenR1은 DeepSeek-R1의 방법론을 투명하게 공개하고, OpenR은 연구 프레임워크를 제공하며, Qwen3는 thinking/non-thinking 통합이라는 실용적 혁신을 이루었다. 이러한 오픈소스 모델들은 상업 모델과의 성능 격차를 빠르게 좁히며, 추론 능력의 민주화에 기여하고 있습니다.
