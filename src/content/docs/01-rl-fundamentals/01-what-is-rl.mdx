## 강화학습의 직관적 이해

강화학습(Reinforcement Learning, RL)은 **에이전트(agent)** 가 **환경(environment)** 과 상호작용하면서 시행착오(trial and error)를 통해 최적의 행동 전략을 학습하는 머신러닝 패러다임입니다. 지도학습처럼 정답 레이블이 주어지지 않으며, 비지도학습처럼 데이터의 구조를 찾는 것도 아닙니다. 에이전트는 오직 **보상 신호(reward signal)** 만을 통해 자신의 행동이 얼마나 좋았는지를 평가받습니다.

체스를 두는 AI를 예로 들어보겠습니다. 매 수마다 "이 수가 좋다/나쁘다"는 레이블이 없습니다. 게임이 끝날 때 승패라는 보상만 존재합니다. 에이전트는 수천, 수백만 번의 대국을 통해 어떤 상황에서 어떤 수를 두어야 이길 확률이 높은지를 스스로 발견해야 합니다.

강화학습의 역사는 1950년대 동물 행동 심리학(조작적 조건화)에서 시작합니다. Bellman(1957)의 동적 프로그래밍, Samuel(1959)의 체커 프로그램을 거쳐, Sutton & Barto(1988)의 TD 학습이 이론적 기반을 확립했습니다. 2013년 DeepMind의 DQN이 아타리 게임에서 인간 수준의 성능을 달성하며 딥 강화학습의 시대가 열렸고, 이후 AlphaGo(2016), ChatGPT의 RLHF(2022)로 이어졌습니다.

## 마르코프 결정 과정 (MDP)

강화학습의 수학적 기반은 **마르코프 결정 과정(Markov Decision Process, MDP)** 입니다. MDP는 다음 5개의 구성요소로 정의됩니다:

| 기호 | 명칭 | 설명 |
|------|------|------|
| $\mathcal{S}$ | 상태 공간(State Space) | 환경이 취할 수 있는 모든 상태의 집합 |
| $\mathcal{A}$ | 행동 공간(Action Space) | 에이전트가 선택할 수 있는 모든 행동의 집합 |
| $P$ | 전이 함수(Transition Function) | $P(s' \mid s, a)$: 상태 $s$에서 행동 $a$를 했을 때 상태 $s'$로 전이될 확률 |
| $R$ | 보상 함수(Reward Function) | $R(s, a, s')$: 상태 전이에 따른 즉각적인 보상 |
| $\gamma$ | 할인 인자(Discount Factor) | $\gamma \in [0, 1]$: 미래 보상의 현재 가치 |

MDP의 핵심 가정은 **마르코프 성질(Markov Property)** 입니다. 현재 상태 $s_t$가 미래를 결정하는 데 필요한 모든 정보를 담고 있다는 것입니다. 즉, 과거 상태의 전체 이력을 몰라도 현재 상태만으로 충분합니다:

$$P(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} \mid s_t, a_t)$$

## GridWorld: MDP의 구체적 예시

**GridWorld** 는 강화학습을 설명하는 가장 대표적인 예제입니다. 4×4 격자 세계를 통해 MDP의 모든 구성요소를 직접 확인할 수 있습니다.

```
┌───┬───┬───┬───┐
│ S │   │   │   │
├───┼───┼───┼───┤
│   │ X │   │   │
├───┼───┼───┼───┤
│   │   │   │   │
├───┼───┼───┼───┤
│   │   │   │ G │
└───┴───┴───┴───┘
S: 시작점  X: 장애물  G: 목표(보상 +1)
```

이 GridWorld를 MDP로 매핑하면:

- **상태 공간 $\mathcal{S}$**: 16개 격자 위치 (장애물 제외 15개 유효 상태)
- **행동 공간 $\mathcal{A}$**: 상/하/좌/우 4가지
- **전이 함수 $P$**: 결정적(deterministic) — 선택한 방향으로 이동. 단, 벽에 부딪히면 제자리
- **보상 함수 $R$**: 목표(G)에 도달하면 $+1$, 그 외 모든 전이에서 $-0.04$ (경로 최소화 유도)
- **할인 인자 $\gamma$**: 0.9 (근미래 선호)

```python
# GridWorld MDP 예시 (Python 의사코드)
class GridWorldMDP:
    def __init__(self, size=4):
        self.size = size
        self.goal = (3, 3)
        self.obstacle = (1, 1)
        self.gamma = 0.9

    def transition(self, state, action):
        """결정적 전이: 다음 상태 반환"""
        row, col = state
        moves = {'up': (-1,0), 'down': (1,0),
                 'left': (0,-1), 'right': (0,1)}
        dr, dc = moves[action]
        new_state = (max(0, min(self.size-1, row+dr)),
                     max(0, min(self.size-1, col+dc)))
        # 장애물이면 제자리
        if new_state == self.obstacle:
            new_state = state
        return new_state

    def reward(self, state, next_state):
        if next_state == self.goal:
            return 1.0
        return -0.04  # 매 스텝 소량 패널티
```

GridWorld에서 최적 경로는 장애물을 우회하며 목표에 도달하는 것입니다. 할인 인자가 낮을수록 에이전트는 즉각적인 보상을 추구하여 더 짧은 경로를 선택하려 합니다.

## 핵심 직관: 마르코프 성질이 왜 중요한가?

> **마르코프 성질은 상태 설계의 핵심입니다.** 상태가 마르코프 성질을 만족하지 않으면, 에이전트는 과거 이력을 기억해야 하므로 상태 공간이 폭발적으로 커집니다.

체스를 예로 들면, 현재 보드 배치(상태)만 알면 이후 최적 전략을 결정할 수 있습니다. 어떤 수순으로 현재 배치에 도달했는지는 중요하지 않습니다 — 이것이 마르코프 성질입니다. LLM의 경우, 현재까지 생성된 토큰 시퀀스 전체가 상태를 구성하며, 이 시퀀스가 다음 토큰 생성에 필요한 모든 정보를 담고 있습니다.

## 에이전트-환경 상호작용 루프

에이전트와 환경의 상호작용은 다음과 같은 사이클로 진행됩니다:

1. 에이전트가 현재 상태 $s_t$를 관측합니다
2. 에이전트가 정책에 따라 행동 $a_t$를 선택합니다
3. 환경이 전이 함수 $P$에 따라 다음 상태 $s_{t+1}$로 전이합니다
4. 환경이 보상 $r_t = R(s_t, a_t, s_{t+1})$을 에이전트에게 전달합니다
5. $t \leftarrow t + 1$, 1단계로 돌아갑니다

이 루프는 에피소드가 종료(terminal state 도달)되거나, 연속 과제의 경우 무한히 반복됩니다.

## 에피소드 과제 vs 연속 과제

| 특성 | 에피소드 과제 (Episodic) | 연속 과제 (Continuing) |
|------|--------------------------|------------------------|
| 종료 상태 | 존재 (terminal state) | 없음 |
| 할인 인자 | $\gamma \leq 1$ (보통 1도 가능) | $\gamma < 1$ 필수 |
| 보상 합 | 유한 에피소드 내에서 수렴 | $\gamma < 1$이어야 수렴 |
| 대표 예시 | 바둑, 아타리 게임, 대화 생성 | 로봇 제어, 주식 거래 |
| 학습 단위 | 에피소드 전체 | 연속적 시간 윈도우 |
| LLM 해당 여부 | **해당** (프롬프트→응답 1회) | 해당 없음 |

**에피소드 과제(Episodic Task)** 는 명확한 종료 상태가 존재합니다. LLM의 응답 생성은 EOS(End-of-Sequence) 토큰 생성으로 에피소드가 종료되는 전형적인 에피소드 과제입니다.

**연속 과제(Continuing Task)** 는 종료 상태가 없습니다. 이 경우 보상의 합이 무한대가 될 수 있으므로 할인 인자가 반드시 필요합니다.

## 누적 보상과 할인 인자

에이전트의 목표는 시간 $t$부터 받을 **누적 보상(Return)** $G_t$를 최대화하는 것입니다:

$$G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$$

이 무한 급수가 수렴하려면 $\gamma < 1$ 또는 보상이 유한 시점에서 끝나야 합니다. $\gamma < 1$일 때 상한은:

$$G_t \leq \frac{R_{max}}{1 - \gamma}$$

예를 들어 $R_{max} = 1$, $\gamma = 0.99$이면 $G_t \leq 100$으로 유한합니다.

**할인 인자 $\gamma$의 직관적 해석:**

- $\gamma = 0$: 완전한 근시안 — 오직 즉각 보상 $R_{t+1}$만 고려
- $\gamma = 0.9$: 10스텝 후 보상은 현재 가치의 $0.9^{10} \approx 35\%$로 할인
- $\gamma = 0.99$: 100스텝 후 보상도 $0.99^{100} \approx 37\%$ 가치 유지
- $\gamma \to 1$: 원시안 — 먼 미래 보상도 현재와 동등하게 취급

Sutton & Barto (2018)에 따르면, $\gamma$는 학습 문제의 특성에 맞게 설계되어야 합니다. LLM의 대화 생성에서는 보통 $\gamma = 1$ (에피소드 과제)이거나, 각 토큰 생성에 매우 작은 보상을 부여하는 경우에는 $\gamma < 1$이 사용됩니다.

## 최적성의 기준

강화학습에서 "최적(optimal)"의 기준은 기대 누적 보상의 최대화입니다. 에이전트가 정책 $\pi$를 따를 때의 기대 누적 보상은:

$$J(\pi) = \mathbb{E}_\pi \left[ G_0 \right] = \mathbb{E}_\pi \left[ \sum_{t=0}^{\infty} \gamma^t R_{t+1} \right]$$

이 값을 최대화하는 정책 $\pi^*$를 **최적 정책(optimal policy)** 이라 합니다.

GridWorld 예시에서 최적 정책은 "목표에 최단 경로로 도달하되 장애물을 우회"하는 전략입니다. 이 최적 정책을 어떻게 표현하고 학습하는지가 다음 장의 주제인 정책과 가치 함수 개념으로 이어집니다. 이 개념은 **섹션 1-2(정책과 가치 함수)** 에서 더 자세히 다룹니다.
