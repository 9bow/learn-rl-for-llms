## 강화학습의 직관적 이해

강화학습(Reinforcement Learning, RL)은 **에이전트(agent)** 가 **환경(environment)** 과 상호작용하면서 시행착오(trial and error)를 통해 최적의 행동 전략을 학습하는 머신러닝 패러다임입니다. 지도학습처럼 정답 레이블이 주어지지 않으며, 비지도학습처럼 데이터의 구조를 찾는 것도 아닙니다. 에이전트는 오직 **보상 신호(reward signal)** 만을 통해 자신의 행동이 얼마나 좋았는지를 평가받습니다.

체스를 두는 AI를 예로 들어보겠습니다. 매 수마다 "이 수가 좋다/나쁘다"는 레이블이 없습니다. 게임이 끝날 때 승패라는 보상만 존재합니다. 에이전트는 수천, 수백만 번의 대국을 통해 어떤 상황에서 어떤 수를 두어야 이길 확률이 높은지를 스스로 발견해야 합니다.

## 마르코프 결정 과정 (MDP)

강화학습의 수학적 기반은 **마르코프 결정 과정(Markov Decision Process, MDP)** 입니다. MDP는 다음 5개의 구성요소로 정의됩니다:

| 기호 | 명칭 | 설명 |
|------|------|------|
| $\mathcal{S}$ | 상태 공간(State Space) | 환경이 취할 수 있는 모든 상태의 집합 |
| $\mathcal{A}$ | 행동 공간(Action Space) | 에이전트가 선택할 수 있는 모든 행동의 집합 |
| $P$ | 전이 함수(Transition Function) | $P(s' \mid s, a)$: 상태 $s$에서 행동 $a$를 했을 때 상태 $s'$로 전이될 확률 |
| $R$ | 보상 함수(Reward Function) | $R(s, a, s')$: 상태 전이에 따른 즉각적인 보상 |
| $\gamma$ | 할인 인자(Discount Factor) | $\gamma \in [0, 1]$: 미래 보상의 현재 가치 |

MDP의 핵심 가정은 **마르코프 성질(Markov Property)** 입니다. 현재 상태 $s_t$가 미래를 결정하는 데 필요한 모든 정보를 담고 있다는 것입니다. 즉, 과거 상태의 전체 이력을 몰라도 현재 상태만으로 충분합니다:

$$P(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} \mid s_t, a_t)$$

## 에이전트-환경 상호작용 루프

에이전트와 환경의 상호작용은 다음과 같은 사이클로 진행됩니다:

1. 에이전트가 현재 상태 $s_t$를 관측합니다
2. 에이전트가 정책에 따라 행동 $a_t$를 선택합니다
3. 환경이 전이 함수 $P$에 따라 다음 상태 $s_{t+1}$로 전이합니다
4. 환경이 보상 $r_t = R(s_t, a_t, s_{t+1})$을 에이전트에게 전달합니다
5. $t \leftarrow t + 1$, 1단계로 돌아갑니다

이 루프는 에피소드가 종료(terminal state 도달)되거나, 연속 과제의 경우 무한히 반복됩니다.

## 에피소드 과제 vs 연속 과제

**에피소드 과제(Episodic Task)** 는 명확한 종료 상태가 존재합니다. 바둑, 아타리 게임, 대화 생성 등이 여기에 해당합니다. 각 에피소드는 독립적으로 시작되며, 에피소드 내의 모든 경험이 하나의 학습 단위가 됩니다.

**연속 과제(Continuing Task)** 는 종료 상태가 없습니다. 로봇 제어, 주식 거래 시스템 등이 해당됩니다. 이 경우 보상의 합이 무한대가 될 수 있으므로 할인 인자가 반드시 필요합니다.

## 누적 보상과 할인 인자

에이전트의 목표는 시간 $t$부터 받을 **누적 보상(Return)** $G_t$를 최대화하는 것입니다:

$$G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$$

할인 인자 $\gamma$는 두 가지 역할을 합니다:

- **수학적 역할**: $\gamma < 1$이면 무한 합이 수렴하여 $G_t \leq R_{max}/(1-\gamma)$로 유한한 값이 됩니다
- **경제적 역할**: 미래의 보상은 현재의 보상보다 덜 가치 있다는 직관을 반영합니다. $\gamma = 0$이면 근시안적(즉각 보상만 고려), $\gamma \to 1$이면 원시안적(먼 미래까지 고려)

Sutton & Barto (2018)에 따르면, $\gamma$는 학습 문제의 특성에 맞게 설계되어야 합니다. LLM의 대화 생성에서는 보통 $\gamma = 1$ (에피소드 과제)이거나, 각 토큰 생성에 매우 작은 보상을 부여하는 경우에는 $\gamma < 1$이 사용됩니다.

## 최적성의 기준

강화학습에서 "최적(optimal)"의 기준은 기대 누적 보상의 최대화입니다. 에이전트가 정책 $\pi$를 따를 때의 기대 누적 보상은:

$$J(\pi) = \mathbb{E}_\pi \left[ G_0 \right] = \mathbb{E}_\pi \left[ \sum_{t=0}^{\infty} \gamma^t R_{t+1} \right]$$

이 값을 최대화하는 정책 $\pi^*$를 **최적 정책(optimal policy)** 이라 합니다. 다음 장에서는 이 최적 정책을 어떻게 표현하고 학습하는지를 다루는 정책과 가치 함수 개념으로 나아갑니다.
