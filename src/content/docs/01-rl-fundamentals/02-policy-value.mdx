## 정책(Policy)의 정의

**정책(policy)** [\\(\pi(a \mid s)\\)](../00-notation/#policies)는 에이전트의 행동 전략을 정의합니다. 각 상태에서 어떤 행동을 선택할지를 결정하는 함수입니다.

**결정적 정책(Deterministic Policy)** 은 각 상태에 정확히 하나의 행동을 매핑합니다:

$$\pi(s) = a$$

> **직관.** 같은 상태에 들어가면 언제나 똑같은 행동 하나만 나오므로, 탐색 여지가 전혀 없는 규칙표와 같은 역할을 합니다.

**확률적 정책(Stochastic Policy)** 은 각 상태에서 행동에 대한 확률 분포를 출력합니다:

$$\pi(a \mid s) = P(A_t = a \mid S_t = s)$$

> **직관.** 같은 상태라도 매번 다른 행동이 뽑힐 수 있어서, 확률의 크기가 곧 그 행동을 얼마나 선호하는지를 결정하고 자연스러운 탐색을 만들어 냅니다.

LLM의 텍스트 생성은 본질적으로 확률적 정책입니다. 각 상태(현재까지 생성된 토큰 시퀀스)에서 다음 토큰(행동)의 확률 분포를 출력합니다. 확률적 정책은 탐색(exploration)을 자연스럽게 내포하므로, 강화학습에서 더 광범위하게 활용됩니다.

## 상태 가치 함수 V^π(s)

**상태 가치 함수(State Value Function)** [\\(V^\pi\\)](../00-notation/#value-functions)는 정책 $\pi$를 따를 때 상태 $s$에서 시작하여 얻을 수 있는 기대 누적 보상입니다. 여기서 할인 계수 [\\(\gamma\\)](../00-notation/#rewards-and-returns)는 미래 보상을 현재 가치로 얼마만큼 끌어당길지를 정합니다:

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid S_t = s \right] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t = s \right]$$

> **직관.** 현재 자리에서 출발해 이 정책으로 끝까지 굴렸을 때 기대되는 점수로, 미래 보상일수록 $\gamma$ 만큼 지수적으로 작게 반영되어 "가까운 미래가 더 중요하다"는 의미를 담습니다.

이 함수는 "현재 상태가 얼마나 좋은가?"를 측정합니다. 가치가 높은 상태는 미래에 높은 보상을 기대할 수 있는 상태입니다.

### 가치 함수 계산 예시: 3×1 GridWorld

간단한 1행 3열 격자를 통해 가치 함수를 직접 계산해 보겠습니다.

```
[ A ] [ B ] [ G ]
  0    1     2
```

- 상태: A(0), B(1), G(목표, 2)
- 행동: 우(right), 좌(left). 목표 G는 종료 상태
- 보상: G에 도달하면 $+1$, 그 외 $0$
- 정책: 항상 우(right)로 이동하는 결정적 정책 $\pi$
- $\gamma = 0.9$

**V^π(A) 계산:**

$$V^\pi(A) = 0 + \gamma \cdot V^\pi(B) = 0 + 0.9 \cdot V^\pi(B)$$

{/* skip-intuition: numerical substitution in worked example */}

**V^π(B) 계산:**

$$V^\pi(B) = 0 + \gamma \cdot V^\pi(G) = 0 + 0.9 \cdot 1.0 = 0.9$$

{/* skip-intuition: numerical substitution in worked example */}

**V^π(A) 최종:**

$$V^\pi(A) = 0.9 \times 0.9 = 0.81$$

{/* skip-intuition: numerical substitution in worked example */}

즉, 목표로부터 2스텝 떨어진 상태 A의 가치는 $\gamma^2 \times 1 = 0.81$이고, 1스텝 거리인 B는 $0.9$입니다. 이 직관은 "목표에 가까울수록 현재 상태의 가치가 높다"는 것을 수치로 확인시켜 줍니다.

## 행동 가치 함수 Q^π(s, a)

**행동 가치 함수(Action Value Function)** [\\(Q^\pi\\)](../00-notation/#value-functions)는 상태 $s$에서 행동 $a$를 선택한 뒤, 이후 정책 $\pi$를 따를 때의 기대 누적 보상입니다:

$$Q^\pi(s, a) = \mathbb{E}_\pi \left[ G_t \mid S_t = s, A_t = a \right]$$

> **직관.** 지금 이 한 번만 특정 행동 $a$를 강제로 취하고 그 후에는 정책대로 따른다고 했을 때 기대되는 점수이며, 행동 선택이 결과를 어떻게 바꾸는지를 직접 드러내는 역할을 합니다.

$V^\pi$와 $Q^\pi$의 관계는 다음과 같습니다:

$$V^\pi(s) = \sum_{a \in \mathcal{A}} \pi(a \mid s) \cdot Q^\pi(s, a)$$

> **직관.** 상태 가치는 그 상태에서 뽑힐 수 있는 여러 행동 가치를 정책 확률로 가중 평균한 값이라, 정책이 어떤 행동에 확률을 몰아주는지에 따라 $V$가 오르내리는 의미를 갖습니다.

즉, 상태 가치는 행동 가치의 정책 가중 평균입니다.

## 벨만 기대 방정식 (Bellman Expectation Equation)

가치 함수의 핵심 성질은 **재귀적(recursive)** 관계입니다. 이를 **벨만 기대 방정식** 이라 합니다:

$$V^\pi(s) = \sum_a \pi(a \mid s) \sum_{s'} P(s' \mid s, a) \left[ R(s, a, s') + \gamma V^\pi(s') \right]$$

> **직관.** 현재 상태 가치를 "바로 받는 보상 + 다음 상태 가치의 할인분"으로 쪼개 주므로, 무한한 미래 합산을 한 스텝 앞만 내다보는 문제로 바꾸는 역할을 합니다.

$$Q^\pi(s, a) = \sum_{s'} P(s' \mid s, a) \left[ R(s, a, s') + \gamma \sum_{a'} \pi(a' \mid s') Q^\pi(s', a') \right]$$

> **직관.** 이번에 선택한 행동이 어떤 다음 상태로 데려갈지를 전이 확률로 가중하고, 거기서 정책이 만들어 낼 다음 $Q$ 값까지 이어 붙여서 행동 선택의 장기 영향을 해석할 수 있게 만듭니다.

이 방정식은 "현재 상태의 가치 = 즉각 보상 + 다음 상태의 할인된 가치"임을 말합니다. 이 재귀 구조는 동적 프로그래밍(Dynamic Programming)을 통한 가치 계산의 기초입니다.

## 핵심 직관: 벨만 방정식이 왜 강력한가?

> **벨만 방정식은 시간적으로 분리된 최적화 문제를 시간 단계별로 분해합니다.** 전체 에피소드의 누적 보상을 한꺼번에 최적화하는 대신, 매 스텝마다 "즉각 보상 + 다음 상태 가치"라는 더 단순한 문제를 반복 풀 수 있습니다. 이것이 강화학습 알고리즘의 계산 가능성을 만드는 핵심입니다.

## 벨만 최적 방정식 (Bellman Optimality Equation)

최적 정책 $\pi^*$에 대응하는 **최적 가치 함수** $V^*(s)$와 $Q^*(s,a)$는:

$$V^*(s) = \max_a \sum_{s'} P(s' \mid s, a) \left[ R(s, a, s') + \gamma V^*(s') \right]$$

> **직관.** 기대 방정식의 $\sum_a \pi(a \mid s)$ 평균 대신 $\max_a$ 가 자리 잡아, "행동들을 섞는 대신 가장 좋은 하나에 모든 확률을 몰아준다"는 의미로 바뀌고 그 결과가 곧 최적 가치가 됩니다.

$$Q^*(s, a) = \sum_{s'} P(s' \mid s, a) \left[ R(s, a, s') + \gamma \max_{a'} Q^*(s', a') \right]$$

> **직관.** 다음 상태에서는 어떻게든 가장 좋은 행동을 집어낸다고 가정하므로, 현재 행동 $a$의 실력은 "최선의 미래를 전제했을 때의 기대 수익"으로 해석됩니다.

최적 정책은 각 상태에서 $Q^*$ 값이 가장 높은 행동을 선택합니다:

$$\pi^*(s) = \arg\max_a Q^*(s, a)$$

> **직관.** $Q^*$ 표가 완성되기만 하면 별도의 학습 없이 각 상태에서 값이 가장 큰 행동만 읽어 오면 되므로, 가치 함수를 정책으로 바꿔 주는 결정 규칙 역할을 합니다.

**최적 정책의 존재성**: 유한 MDP에서는 항상 결정적인 최적 정책이 존재합니다(Sutton & Barto, 2018, Chapter 3). 이는 강화학습의 근본적인 이론적 보장입니다.

## 정책 반복 vs 가치 반복

MDP를 풀기 위한 두 가지 대표적인 동적 프로그래밍 알고리즘을 비교합니다.

### 정책 반복 (Policy Iteration)

정책 평가(Policy Evaluation)와 정책 개선(Policy Improvement)을 교대로 수행합니다:

```python
def policy_iteration(mdp, gamma=0.9, tol=1e-6):
    # 임의 정책 초기화
    policy = {s: random_action(s) for s in mdp.states}
    V = {s: 0.0 for s in mdp.states}

    while True:
        # 1단계: 정책 평가 (현재 정책의 가치 계산)
        while True:
            delta = 0
            for s in mdp.states:
                a = policy[s]
                v_new = sum(
                    mdp.P(s_next, s, a) * (mdp.R(s, a, s_next) + gamma * V[s_next])
                    for s_next in mdp.states
                )
                delta = max(delta, abs(v_new - V[s]))
                V[s] = v_new
            if delta < tol:
                break  # 수렴

        # 2단계: 정책 개선 (greedy update)
        policy_stable = True
        for s in mdp.states:
            old_action = policy[s]
            policy[s] = argmax over a of Q(s, a)
            if old_action != policy[s]:
                policy_stable = False

        if policy_stable:
            return policy, V  # 최적 정책 도달
```

### 가치 반복 (Value Iteration)

정책 평가를 1회만 수행하고 즉시 정책 개선을 적용합니다:

```python
def value_iteration(mdp, gamma=0.9, tol=1e-6):
    V = {s: 0.0 for s in mdp.states}

    while True:
        delta = 0
        for s in mdp.states:
            # 벨만 최적 방정식을 한 번에 적용
            v_new = max(
                sum(
                    mdp.P(s_next, s, a) * (mdp.R(s, a, s_next) + gamma * V[s_next])
                    for s_next in mdp.states
                )
                for a in mdp.actions
            )
            delta = max(delta, abs(v_new - V[s]))
            V[s] = v_new

        if delta < tol:
            break

    # 최적 정책 추출
    policy = {s: argmax_a Q_star(s, a) for s in mdp.states}
    return policy, V
```

| 특성 | 정책 반복 | 가치 반복 |
|------|----------|----------|
| 수렴 속도 | 빠름 (반복 횟수 적음) | 느림 (반복 횟수 많음) |
| 매 반복 비용 | 높음 (정책 평가 수렴까지) | 낮음 (벨만 최적 1회 적용) |
| 구현 복잡도 | 중간 | 단순 |
| 수렴 보장 | 유한 MDP에서 보장 | 유한 MDP에서 보장 |
| LLM 적용 가능성 | 상태 공간이 너무 커 직접 적용 불가 | 상태 공간이 너무 커 직접 적용 불가 |

유한 소규모 MDP에서는 두 알고리즘 모두 최적 정책을 보장하지만, LLM처럼 상태 공간이 지수적으로 큰 경우에는 함수 근사(deep neural network)가 필요합니다.

## 가치 기반 vs 정책 기반 방법 비교

| 특성 | 가치 기반 (Value-Based) | 정책 기반 (Policy-Based) |
|------|------------------------|------------------------|
| 대표 알고리즘 | Q-learning, DQN | REINFORCE, PPO |
| 학습 대상 | $Q^*(s, a)$ 또는 $V^*(s)$ | $\pi_\theta(a \mid s)$ |
| 정책 도출 | $\pi(s) = \arg\max_a Q(s, a)$ | 직접 출력 |
| 행동 공간 | 이산 공간에 적합 | 연속/이산 모두 가능 |
| 수렴 안정성 | 함수 근사 시 불안정 가능 | 상대적으로 안정적 |
| 분산 | 낮음 | 높음 |
| 편향 | 높음 (Q 추정 오차) | 낮음 |
| LLM 적용 | 토큰 수준 Q값 추정 어려움 | 자연스러운 매핑 |

LLM 정렬 분야에서는 행동 공간(전체 어휘, 수만 개의 토큰)이 매우 크기 때문에, $Q^*$를 직접 학습하는 가치 기반 방법보다 정책 기반 방법(PPO 등)이 주로 사용됩니다. DQN(Mnih et al., 2015)이 아타리 게임에서 성공한 것은 이산적이고 제한된 행동 공간(18개 행동) 덕분이었습니다.

## Actor-Critic: 두 세계의 결합

**Actor-Critic** 방법은 가치 기반과 정책 기반의 장점을 결합합니다. Actor(정책)가 행동을 선택하고, Critic(가치 함수)이 그 행동의 품질을 평가하여 Actor의 학습을 돕습니다.

Actor-Critic의 핵심 이점은 Critic이 제공하는 **어드밴티지(Advantage)** [\\(A^\pi\\)](../00-notation/#value-functions) 추정치입니다:

$$A^\pi(s, a) = Q^\pi(s, a) - V^\pi(s)$$

> **직관.** 상태 평균값 $V^\pi$를 기준선으로 빼 주므로, 같은 상태에서 행동 사이의 상대 우열만 남아 정책 경사 신호의 분산을 크게 줄이는 역할을 합니다.

어드밴티지는 "이 행동이 평균(기대값)보다 얼마나 좋은가?"를 측정합니다. 양수 어드밴티지는 이 행동이 평균보다 좋음을, 음수는 평균보다 나쁨을 의미합니다. 이를 통해 정책 경사의 분산을 크게 줄일 수 있습니다.

이 구조는 다음 장에서 다루는 PPO의 기반이 되며, RLHF에서 핵심 역할을 합니다. 정책과 가치 함수의 관계는 **섹션 1-3(정책 경사법)** 과 **섹션 1-4(Actor-Critic과 PPO)** 에서 더 자세히 다룹니다.
