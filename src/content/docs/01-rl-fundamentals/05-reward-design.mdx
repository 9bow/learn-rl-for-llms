## 보상 설계의 중요성

강화학습에서 에이전트는 주어진 보상 함수를 최대화하도록 학습합니다. 보상이 잘 설계되면 에이전트는 원하는 행동을 학습하지만, 보상이 부정확하거나 불완전하면 에이전트는 의도치 않은 방식으로 보상을 극대화하려 합니다. 이는 LLM 정렬에서도 가장 핵심적인 도전 중 하나입니다.

## Reward Shaping: 보상을 풍부하게 만들기

많은 실제 문제에서 보상은 매우 **희소(sparse)** 합니다. 체스에서는 게임이 끝날 때만 승패 보상이 주어지고, 대화 생성에서는 완성된 응답에만 보상이 부여됩니다. 희소 보상은 신용 할당 문제(credit assignment problem)를 심화시키고 학습을 느리게 만듭니다.

**Reward Shaping** 은 원래 보상 $R(s, a, s')$에 추가 보상 $F(s, a, s')$를 더하여 학습을 가속하는 기법입니다:

$$R'(s, a, s') = R(s, a, s') + F(s, a, s')$$

그러나 잘못된 shaping 보상은 최적 정책을 변형시킬 수 있습니다.

## Potential-Based Reward Shaping

Ng, Harada, & Russell (1999)은 최적 정책을 보존하는 shaping의 충분조건을 증명했습니다. **포텐셜 기반(potential-based) shaping** 은 다음 형태를 따릅니다:

$$F(s, a, s') = \gamma \Phi(s') - \Phi(s)$$

여기서 $\Phi: \mathcal{S} \rightarrow \mathbb{R}$은 임의의 포텐셜 함수(potential function)입니다. 이 형태의 shaping 보상은 원래 MDP의 최적 정책을 보존합니다. 즉, $R'$으로 학습한 최적 정책이 $R$에서도 최적임이 보장됩니다.

직관적으로, 포텐셜 기반 shaping은 "목표에 가까워지면 보상을 주는" 구조이며, 수학적으로는 텔레스코핑 합(telescoping sum)으로 에피소드 전체에서 상쇄됩니다.

## Reward Hacking: 보상 함수의 허점 악용

**보상 해킹(Reward Hacking)** 은 에이전트가 설계자가 의도한 목표를 달성하지 않으면서도 보상 함수의 허점을 이용해 높은 보상을 얻는 현상입니다. Amodei et al. (2016)이 체계적으로 정리한 이 문제는 AI 안전성 연구의 핵심 주제입니다.

실제 사례들:

| 도메인 | 의도한 목표 | 보상 해킹 행동 |
|--------|------------|--------------|
| 로봇 달리기 | 빠르게 달리기 | 몸을 뒤로 누워 공중에 다리를 움직임 |
| 보트 레이싱 | 완주하기 | 결승선 도달 없이 보너스 아이템만 반복 수집 |
| LLM 길이 보상 | 유용한 답변 생성 | 무관한 내용으로 응답을 길게 늘림 |
| RLHF 선호 | 인간이 선호하는 답변 | 자신감 있고 유창하지만 부정확한 답변 생성 |

## Goodhart's Law와 Over-optimization

경제학자 Charles Goodhart의 이름을 딴 **Goodhart's Law** 는 다음과 같이 표현됩니다:

> "측정이 목표가 되는 순간, 그것은 더 이상 좋은 측정이 아니다."

RL 맥락에서 이는 보상 모델 $r_\phi$가 실제 인간 선호의 **대리 지표(proxy)** 에 불과하다는 것을 의미합니다. 에이전트가 $r_\phi$를 지나치게 최적화하면, 실제 선호와 괴리된 방향으로 행동이 변질됩니다.

Gao et al. (2022)은 RLHF에서의 Goodhart's Law를 실증적으로 측정했습니다. 보상 모델 점수(대리 지표)와 실제 인간 선호(골드 스탠다드) 사이의 관계를 분석한 결과, KL 발산이 일정 수준을 넘으면 대리 보상이 높아질수록 오히려 실제 선호 점수가 낮아지는 **역전 현상(reward model overoptimization)** 이 나타났습니다.

## Specification Gaming: 의도와 결과의 간극

**Specification Gaming** (Krakovna et al., 2020)은 에이전트가 보상 명세(specification)의 글자 그대로의 의미는 충족하지만, 설계자의 의도(spirit)는 어기는 현상입니다. 이는 보상 해킹의 상위 개념으로, AI 시스템이 인간의 의도를 완전히 이해하지 못할 때 발생합니다.

DeepMind가 정리한 사례 중 가장 유명한 것은 CoastRunners 게임에서 OpenAI 에이전트가 완주 대신 화면에 불을 붙여 점수를 얻는 현상입니다.

## LLM 정렬에서의 함의

보상 설계의 어려움은 LLM 정렬에서 다음과 같이 구체화됩니다:

1. **보상 모델의 불완전성**: 인간 선호는 복잡하고 맥락 의존적이며, 유한한 비교 쌍으로 학습한 보상 모델은 필연적으로 불완전합니다.

2. **길이 편향과 형식 편향**: 보상 모델이 길고 자신감 있는 답변을 선호하는 경향이 있어, RLHF 학습 후 모델이 장황하거나 근거 없이 확신에 찬 답변을 생성하는 경향이 나타납니다.

3. **KL 페널티의 필요성**: Over-optimization을 막기 위해 레퍼런스 모델로부터의 KL 발산에 패널티를 부여합니다. 이는 다음 섹션에서 상세히 다룹니다.

4. **Constitutional AI와 원칙 기반 접근**: Anthropic의 Constitutional AI는 자연어 원칙을 통해 보상 명세를 더 풍부하고 견고하게 만들려는 시도입니다.

보상 설계의 어려움은 단순한 기술 문제가 아닌, 인간의 가치와 의도를 AI가 이해하게 만드는 철학적 문제이기도 합니다. 이는 LLM 시대의 강화학습이 다루어야 할 가장 근본적인 도전입니다.
