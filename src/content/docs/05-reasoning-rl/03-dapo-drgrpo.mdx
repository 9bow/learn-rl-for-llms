## GRPO의 실용적 한계

GRPO는 Critic을 제거하여 메모리 효율성을 높였지만, 대규모 추론 훈련에서 몇 가지 실용적 문제가 드러났습니다.

**엔트로피 붕괴(entropy collapse)**: 학습이 진행될수록 모델이 특정 응답 패턴에 수렴하여 다양성이 급감합니다. 수학 문제에서 초기에 발견한 특정 풀이 전략에 고착되어 더 효율적인 방법을 탐색하지 못하는 현상입니다.

**훈련 불안정성**: 보상 분산이 크거나 그룹 크기가 작을 때 advantage 추정이 불안정합니다. 그룹 전체가 정답이거나 전체가 오답인 경우 분산이 0이 되어 학습 신호가 사라집니다.

**길이 편향**: 시퀀스 수준 정규화는 짧은 응답과 긴 응답을 동등하게 취급하지 않아 길이에 따른 보상 편향이 생깁니다.

## DAPO: Decoupled Clip and Dynamic Sampling Policy Optimization

**DAPO** (ByteDance Seed, 2025, arXiv:2503.14476)는 네 가지 핵심 기법으로 위 문제들을 해결합니다.

### 기법 1: Clip-Higher (비대칭 클리핑)

표준 GRPO는 PPO와 동일한 대칭적 클리핑을 사용합니다.

$$
\text{clip}(\rho, 1-\varepsilon, 1+\varepsilon)
$$

DAPO는 이를 비대칭적으로 분리합니다. 양의 advantage(좋은 응답)와 음의 advantage(나쁜 응답)에 서로 다른 클리핑 범위를 적용합니다.

$$
r_{\text{clipped}} = \begin{cases}
\min(\rho \hat{A}, (1 + \varepsilon_{\text{high}}) \hat{A}) & \text{if } \hat{A} > 0 \\
\max(\rho \hat{A}, (1 - \varepsilon_{\text{low}}) \hat{A}) & \text{if } \hat{A} < 0
\end{cases}
$$

$\varepsilon_{\text{high}} > \varepsilon_{\text{low}}$로 설정하면(예: $\varepsilon_{\text{high}} = 0.28$, $\varepsilon_{\text{low}} = 0.20$), 좋은 응답에 대해서는 더 큰 업데이트를 허용하고, 나쁜 응답 억제에는 더 보수적으로 접근합니다. 이는 엔트로피를 유지하고 다양한 탐색을 장려합니다.

**직관**: 기존 GRPO는 좋은 응답과 나쁜 응답에 동일한 강도로 반응합니다. 그러나 좋은 응답을 더 적극적으로 강화하면서 나쁜 응답을 천천히 억제하는 것이 엔트로피 붕괴를 막는 데 효과적입니다.

### 기법 2: Dynamic Sampling (동적 샘플링)

표준 GRPO는 모든 프롬프트에서 동일한 수의 응답을 샘플링합니다. 이때 그룹 내 모든 응답이 동일한 결과(모두 정답 또는 모두 오답)인 경우 advantage의 분산이 0이 되어 학습 신호가 전혀 없습니다.

DAPO는 이를 해결하기 위해 **유효 프롬프트**만 학습에 사용합니다.

$$
\mathcal{D}_{\text{effective}} = \{x \mid \text{Var}(r_1, r_2, \ldots, r_G) > 0\}
$$

즉, 그룹 내 보상에 분산이 있는 프롬프트만 배치에 포함시킵니다. 제거된 프롬프트 대신 다른 프롬프트를 샘플링하여 배치 크기를 유지합니다.

**효과**: 정보 없는 gradient 업데이트를 방지하고, 배치당 실제 학습 신호의 밀도를 극적으로 향상시킵니다. 특히 모델이 어느 정도 학습된 후 대부분의 쉬운 문제에서 100% 정답률을 보일 때 효과가 두드러집니다.

### 기법 3: Token-level Loss (토큰 수준 손실)

GRPO는 시퀀스 수준에서 손실을 평균하지만, 이는 짧은 응답이 긴 응답보다 단위 토큰당 더 큰 gradient 기여를 하게 만드는 길이 편향을 초래합니다.

DAPO는 전체 토큰을 분모로 사용하여 이 편향을 제거합니다.

$$
L_{\text{DAPO}} = -\frac{\displaystyle\sum_{i=1}^{G}\sum_{t=1}^{|y_i|} \min\!\left(\rho_{i,t}\hat{A}_i,\ \text{clip}(\rho_{i,t})\hat{A}_i\right)}{\displaystyle\sum_{i=1}^{G}|y_i|}
$$

분모가 전체 토큰 수이므로, 각 토큰의 gradient 기여가 응답 길이에 관계없이 균등해집니다. 이는 "짧게 쓰면 더 이득"이라는 편향을 제거하여 모델이 추론에 필요한 만큼 긴 응답을 생성하도록 합니다.

### 기법 4: KL 페널티 제거

DAPO는 KL 페널티 항을 완전히 제거합니다.

$$
L_{\text{GRPO}} = \mathbb{E}\!\left[\ldots\right] - \beta\,\mathbb{D}_{\mathrm{KL}}[\pi_\theta \| \pi_{\text{ref}}] \quad \to \quad L_{\text{DAPO}} = \mathbb{E}\!\left[\ldots\right]
$$

KL 페널티는 이미 클리핑 메커니즘이 정책 이탈을 제어하므로 이론적으로 중복됩니다. 또한 참조 모델을 유지하고 KL을 계산하는 비용이 제거되어 학습이 단순화됩니다.

### DAPO 의사코드

```python
def dapo_update(policy, prompts, G, eps_high=0.28, eps_low=0.20):
    # 1. 동적 샘플링: 유효 프롬프트만 선택
    effective_batch = []
    for x in prompts:
        responses = [policy.sample(x) for _ in range(G)]
        rewards = [verifier(x, y) for y in responses]
        if variance(rewards) > 0:  # 학습 신호 있는 프롬프트만
            effective_batch.append((x, responses, rewards))

    # 2. Advantage 계산 (그룹 정규화)
    total_loss = 0
    total_tokens = 0
    for x, responses, rewards in effective_batch:
        mean_r = mean(rewards)
        std_r = std(rewards) + 1e-8
        advantages = [(r - mean_r) / std_r for r in rewards]

        for y_i, A_i in zip(responses, advantages):
            for t in range(len(y_i)):
                rho = policy.prob(y_i[t]) / old_policy.prob(y_i[t])
                # 3. 비대칭 클리핑 (Clip-Higher)
                if A_i > 0:
                    clipped = min(rho, 1 + eps_high)
                else:
                    clipped = max(rho, 1 - eps_low)
                # 4. 토큰 수준 손실 누적
                total_loss += min(rho * A_i, clipped * A_i)
                total_tokens += 1

    # 토큰 수준 정규화 (길이 편향 제거)
    loss = -total_loss / total_tokens
    loss.backward()
```

## Dr. GRPO: 이론적 Bias 제거

**Dr. GRPO** (Liu et al., 2025, arXiv:2503.20783)는 GRPO의 이론적 문제를 정밀하게 분석하고 두 가지 체계적 bias를 식별합니다.

### 분석 1: KL 항의 이론적 불필요성

GRPO의 손실 함수는 다음과 같습니다.

$$
L_{\text{GRPO}} = \underbrace{L_{\text{clip}}}_{\text{PPO 클리핑}} - \beta \underbrace{\mathbb{D}_{\mathrm{KL}}[\pi_\theta \| \pi_{\text{ref}}]}_{\text{중복 항}}
$$

클리핑 메커니즘이 이미 $\rho \in [1-\varepsilon, 1+\varepsilon]$ 범위를 강제하므로, KL 페널티는 이 범위 내에서만 추가 제약을 가합니다. Dr. GRPO는 KL 항이 없을 때 gradient 추정의 분산이 줄어들고 수렴이 빨라짐을 이론적으로 증명했습니다.

### 분석 2: 그룹 정규화의 Length Bias

GRPO의 advantage 추정은 다음과 같습니다.

$$
\hat{A}_i = \frac{r_i - \bar{r}}{\hat{\sigma}_r}, \quad \hat{\sigma}_r = \sqrt{\frac{1}{G}\sum_{i=1}^{G}(r_i - \bar{r})^2}
$$

이 추정은 두 가지 bias를 포함합니다.

**편향된 추정량(biased estimator)**: 샘플 표준편차 $\hat{\sigma}$는 모집단 표준편차 $\sigma$의 편향된 추정값입니다. 특히 $G$가 작을 때(4~8) 편향이 심각합니다.

**길이-보상 공변량(length-reward covariance)**: 응답 길이 $|y_i|$와 보상 $r_i$ 사이에 체계적 상관관계가 있을 때, 정규화가 이를 보정하지 않아 길이에 따른 보상 편향이 남습니다.

Dr. GRPO의 편향 수정 추정량:

$$
\hat{A}_i^{\text{corrected}} = \frac{r_i - \bar{r}}{s_r \cdot \sqrt{1 - 1/G}}
$$

여기서 $s_r = \sqrt{\frac{1}{G-1}\sum(r_i - \bar{r})^2}$는 불편(unbiased) 표준편차 추정량입니다.

### GRPO · DAPO · Dr. GRPO 비교

| 항목 | GRPO | DAPO | Dr. GRPO |
|------|------|------|----------|
| 클리핑 방식 | 대칭 ($\varepsilon$ 단일) | 비대칭 ($\varepsilon_{\text{high}} \neq \varepsilon_{\text{low}}$) | 대칭 |
| KL 페널티 | 있음 ($\beta > 0$) | 없음 | 없음 |
| 샘플링 전략 | 고정 | 동적 (유효 프롬프트) | 고정 |
| 손실 집계 | 시퀀스 수준 | 토큰 수준 | 시퀀스 수준 |
| Bias 수정 | 없음 | 부분적 (길이) | 명시적 (이론 기반) |
| 이론적 근거 | 경험적 | 경험적 | 이론적 |
| 주요 기여 | 기초 알고리즘 | 실용적 개선 | 이론적 정당화 |

### 실제 적용 권장 사항

DAPO와 Dr. GRPO는 상호 보완적입니다. 실무에서 가장 효과적인 조합은 다음과 같습니다.

1. DAPO의 **Dynamic Sampling** — 배치 정보 밀도 최대화
2. DAPO의 **Clip-Higher** — 엔트로피 유지 및 탐색 장려
3. DAPO의 **Token-level Loss** — 길이 편향 제거
4. Dr. GRPO의 **편향 수정 추정량** — 이론적으로 올바른 advantage 계산
5. 공통으로 **KL 항 제거** — 불필요한 제약 및 계산 비용 제거

이 조합은 순수 GRPO 대비 수렴 속도와 최종 성능 모두에서 개선을 보입니다.
