## 자기 학습 추론의 동기

외부 레이블이나 전문가 주석 없이 모델 스스로 추론 능력을 향상시킬 수 있다면 이상적이다. 이 아이디어는 **자기 학습(self-training)** 패러다임으로, 모델이 생성한 올바른 출력을 다시 학습 데이터로 사용하는 방식이다. Eric Zelikman을 중심으로 한 Stanford 그룹은 이 아이디어를 추론에 적용하여 STaR와 Quiet-STaR를 개발했다.

---

## STaR: Self-Taught Reasoner

**STaR** (Zelikman et al., 2022)는 간단하지만 강력한 자기 학습 루프를 제안한다.

### 알고리즘

1. 문제 $x$에 대해 모델이 Chain-of-Thought 추론 $r$과 최종 답 $a$를 생성한다.
2. 정답과 비교하여 **정답에 도달한 경우** 해당 $(x, r, a)$ 쌍을 학습 데이터로 수집한다.
3. 수집된 데이터로 모델을 fine-tuning한다.
4. 반복한다.

### Rationalization 기법

STaR의 핵심 혁신은 **rationalization** 이다. 모델이 직접 추론해서 틀린 경우, 정답 $a^*$를 힌트로 제공하고 모델이 정답에 도달하는 추론 경로를 역으로 생성하도록 한다. 이 rationalized reasoning도 학습 데이터로 사용한다.

$$
r_{\text{rat}} = \text{Model}(x, \text{hint}=a^*)
$$

Rationalization은 학습 데이터의 양을 크게 늘리고, 모델이 어려운 문제에서도 올바른 추론 패턴을 학습하게 한다.

### STaR의 결과

CommonsenseQA에서 GPT-J(6B) 기반 STaR는 few-shot baseline 대비 약 10% 이상의 정확도 향상을 보였으며, 반복 횟수가 늘어날수록 성능이 꾸준히 향상되는 자기 개선 특성을 확인했다.

**STaR의 한계**: 학습 데이터가 성공한 추론에만 한정되므로, 모델이 처음부터 완전히 틀린 문제에서는 학습 신호를 얻지 못한다. 또한 생성된 추론의 다양성이 시간이 지나면서 감소하는 mode collapse 경향이 있다.

## Quiet-STaR: 모든 토큰에서의 내부 사고

**Quiet-STaR** (Zelikman et al., 2024)는 STaR를 근본적으로 확장한다. STaR가 명시적 문제 풀이에 CoT를 적용했다면, Quiet-STaR는 **일반 텍스트 생성의 모든 토큰에서 내부 사고(internal thought)를 생성** 한다.

### 핵심 메커니즘

모든 토큰 위치 $t$에서 모델은 다음을 수행한다.

1. 현재까지의 컨텍스트 $x_{1:t}$를 입력으로 받는다.
2. `<|startofthought|>` 토큰을 시작으로 내부 rationale $r_t$를 생성한다.
3. `<|endofthought|>` 토큰으로 사고를 종료한다.
4. 원래 다음 토큰 $x_{t+1}$을 예측한다.

이 과정에서 사고는 **최종 출력에 포함되지 않는다** ("quiet" = 조용한). 내부 계산에만 사용되어 다음 토큰 예측 품질을 향상시킨다.

### 병렬 사고 생성

Quiet-STaR의 핵심 기술적 기여는 **모든 토큰 위치에서 동시에 사고를 생성하는 병렬화** 다. 순차적으로 처리하면 $O(T)$배 느려지지만, attention mask를 조작하여 모든 위치에서 병렬로 사고를 생성한다.

$$
P(x_{t+1} \mid x_{1:t}) = \alpha \cdot P_{\text{with thought}}(x_{t+1} \mid x_{1:t}, r_t) + (1-\alpha) \cdot P_{\text{base}}(x_{t+1} \mid x_{1:t})
$$

$\alpha$는 사고의 기여도를 조절하는 학습된 스칼라다. 초기에는 기저 모델에 의존하다가, 사고 품질이 향상될수록 사고의 비중이 증가한다.

### 학습: REINFORCE

Quiet-STaR는 REINFORCE로 학습된다. 보상은 **사고 유/무에 따른 다음 토큰 예측 확률의 차이** 다.

$$
R_t = \log P_{\text{with thought}}(x_{t+1}) - \log P_{\text{without thought}}(x_{t+1})
$$

사고가 예측을 개선하면 양의 보상, 악화시키면 음의 보상이다.

### Quiet-STaR의 성과

Mistral-7B 기반 실험에서:

| 벤치마크 | 기저 모델 | Quiet-STaR |
|---------|---------|-----------|
| GSM8K | 5.9% | **10.9%** (+5.0%p) |
| CommonsenseQA | 36.3% | **47.2%** (+10.9%p) |

GSM8K에서 약 2배, CommonsenseQA에서 약 30% 향상이다. 특히 이 개선은 태스크별 fine-tuning 없이 일반 텍스트 학습만으로 달성되었다는 점에서 의미있다.

### Fast Quiet-STaR (EMNLP 2025)

Quiet-STaR의 주요 한계는 추론 지연이다. 모든 토큰에서 사고를 생성하므로 추론 속도가 크게 느려진다. **Fast Quiet-STaR** 는 사고가 실제로 유익한 토큰 위치를 선택적으로 식별하여 필요한 곳에만 사고를 생성함으로써 이 문제를 해결했다.
