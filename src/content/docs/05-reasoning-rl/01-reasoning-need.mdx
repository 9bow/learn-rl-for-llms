## LLM의 추론 한계

대형 언어 모델(LLM)은 자연어 이해, 번역, 요약 등의 태스크에서 인간 수준을 달성했지만, 엄밀한 추론이 필요한 영역에서는 체계적인 실패 패턴을 보입니다. 이 실패는 단순한 지식 부족이 아니라, 모델 아키텍처의 구조적 한계에서 비롯됩니다.

### 수학·논리·코딩 실패 사례

아래 표는 초기 대형 모델들이 주요 벤치마크에서 보인 성능을 요약합니다.

| 벤치마크 | 태스크 유형 | GPT-3 (175B) | 인간 평균 | 실패 원인 |
|---------|-----------|-------------|---------|---------|
| GSM8K | 초등 수학 문장제 | ~5% | ~90% | 다단계 계산 누적 오류 |
| MATH | 경쟁 수학 | ~5% | ~40% | 복잡한 대수·기하 조작 |
| ARC-Challenge | 과학 추론 | ~25% | ~80% | 인과 관계 이해 부족 |
| HumanEval | 파이썬 코딩 | ~0% | ~75% | 알고리즘 설계 실패 |
| LogiQA | 논리 추론 | ~20% | ~86% | 다층 조건 처리 불능 |

**수학 문제**: 단순 산술은 잘 처리하지만, 다단계 계산이나 대수적 조작에서 중간 계산 오류가 누적됩니다. 예를 들어 "철수는 사과 3개를 갖고 있었고, 영희에게 절반을 주었다. 그 후 어머니가 5개를 더 주었다면 총 몇 개인가?"라는 3단계 문제에서 GPT-3는 중간 상태를 추적하지 못해 오류를 범합니다.

**논리 퍼즐**: 삼단논법 같은 단순 논리는 처리하지만, 반사실적 조건(counterfactual)이나 부정 논리가 중첩되면 일관성이 무너집니다. "A가 아니라면 B다, A가 아니다, 그러므로 B다"는 맞히지만, "A가 아니거나 C가 아니라면 B다, A가 아니다, C가 있다, 그러므로 B인가?"처럼 조건이 추가되면 실패율이 급등합니다.

**코딩 과제**: 간단한 함수 작성은 가능하지만 복잡한 알고리즘 설계, 엣지 케이스 처리에서 오류를 범합니다. 더 심각한 문제는 모델이 "자신 있게" 틀린 코드를 생성하는 **과신(overconfidence)** 입니다. 이 코드는 얼핏 보면 합리적으로 보이지만 실행하면 실패합니다.

### 구조적 원인

이러한 실패의 근본 원인은 LLM이 다음 토큰을 예측하는 과정에서 **중간 추론 상태를 명시적으로 유지하지 않기** 때문입니다. 모델은 입력에서 출력으로 단일 순전파(forward pass)로 직접 매핑하며, 복잡한 문제에 필요한 계획(planning)과 검증(verification) 단계가 구조적으로 부재합니다.

$$
P(y \mid x) = \prod_{t=1}^{T} P(y_t \mid x, y_{1:t-1})
$$

위 수식에서 각 토큰 $y_t$는 오직 이전 컨텍스트만을 조건으로 생성됩니다. 중간 계산 결과를 "작업 메모리"에 저장하고 검증하는 메커니즘이 없습니다.

## Chain-of-Thought 프롬프팅

**CoT(Chain-of-Thought)** 프롬프팅은 Wei et al. (2022)이 제안한 방법으로, 모델이 최종 답 이전에 중간 추론 단계를 텍스트로 생성하도록 유도합니다.

### 두 가지 CoT 방식

**Few-shot CoT**: 추론 예시를 프롬프트에 포함합니다.

```
Q: 사탕 12개를 4명이 나누면?
A: 12 ÷ 4 = 3이므로, 한 명당 3개입니다. 답: 3개

Q: 사탕 18개를 6명이 나누면?
A: [모델이 단계별 추론 생성]
```

**Zero-shot CoT**: "Let's think step by step" 혹은 "단계적으로 생각해 봅시다"라는 단일 문구를 추가합니다. Kojima et al. (2022)이 발견한 이 방법은 예시 없이도 상당한 성능 향상을 이끌어냅니다.

### CoT의 성과와 한계

CoT는 수학, 논리, 상식 추론 벤치마크에서 큰 성능 향상을 가져왔습니다. 특히 모델 크기가 커질수록 CoT의 효과가 두드러지는 **창발(emergent)** 현상이 관찰되었습니다. 약 100B 파라미터 이상의 모델에서야 CoT의 이점이 유의미하게 나타나기 시작했습니다.

그러나 CoT는 다음과 같은 한계를 가집니다.

| 한계 | 설명 | 영향 |
|-----|-----|-----|
| 그릇된 추론 경로 | 모델이 틀린 방향으로 추론해도 자기 수정 불가 | 오류 전파 |
| 추론 깊이 제한 | 프롬프트 길이에 제약됨 | 매우 복잡한 문제 한계 |
| 검증 부재 | 생성된 추론 단계의 정오를 확인하지 않음 | 자신감 있는 오답 |
| 창발 의존성 | 소규모 모델에서 효과 없음 | 범용성 부족 |
| 비용 | 더 많은 토큰 생성 필요 | 속도·비용 증가 |

특히 "추론하는 척"하는 문제가 심각합니다. 모델은 형식적으로 올바른 추론 구조를 흉내 내면서도, 실제 논리적 연산은 제대로 수행하지 않을 수 있습니다.

## System 1 vs System 2 사고

심리학자 Daniel Kahneman의 이중 과정 이론에 따르면, 인간의 사고는 두 가지 시스템으로 구성됩니다.

| 특성 | System 1 | System 2 |
|-----|---------|---------|
| 속도 | 빠름 (즉각적) | 느림 (의도적) |
| 처리 방식 | 자동·직관적 | 분석·단계적 |
| 노력 수준 | 노력 없음 | 높은 집중 필요 |
| 예시 | "2+2=?" 즉답 | 복잡한 세금 계산 |
| 오류 패턴 | 편향·휴리스틱 | 복잡도 한계 |
| 병렬성 | 병렬 처리 | 순차 처리 |

표준 LLM은 전형적인 **System 1 기계**입니다. 단일 순전파로 토큰을 생성하며, 어려운 문제와 쉬운 문제에 동일한 계산을 사용합니다. 어려운 수학 문제를 "2+2=?"와 동일한 메커니즘으로 처리합니다.

추론 강화 연구의 핵심 목표는 LLM에 **System 2 능력을 부여**하는 것입니다. 이를 위해 다음이 요구됩니다.

- **추론 토큰 생성**: 최종 답 이전에 명시적 중간 계산 과정 작성
- **자기 검증**: 생성된 추론이 올바른지 스스로 평가
- **백트래킹**: 오류 발견 시 이전 단계로 돌아가 수정
- **적응적 깊이**: 문제 난이도에 따라 추론 길이 자동 조절

## o1 모델과 패러다임 전환

OpenAI의 **o1 모델** (2024)은 이 분야의 패러다임을 바꾼 이정표입니다. o1은 추론 과정에서 긴 내부 "생각(thinking)" 체인을 생성하며, 이 과정에 RL을 적용하여 추론 능력을 강화했습니다.

### o1의 벤치마크 성능

| 벤치마크 | GPT-4o | o1 | o3 (2025) | 의미 |
|---------|-------|-----|---------|-----|
| AIME 2024 | 13.4% | 83.3% | 96.7% | 수학 올림피아드 |
| MATH | 76.6% | 96.4% | ~99% | 경쟁 수학 |
| Codeforces Rating | ~800 | ~1820 | ~2727 | 경쟁 프로그래밍 |
| GPQA Diamond | 53.6% | 77.3% | 87.7% | 대학원 수준 과학 |
| HumanEval | 90.2% | 92.4% | — | 코딩 |

o1에서 o3으로의 도약 역시 주목할 만합니다. 모델 구조 변화보다 추론 시간 계산(test-time compute) 확장이 주요 요인으로 분석됩니다.

o1의 핵심 통찰은 **추론 시간 계산을 확장**하면 성능이 향상된다는 것입니다. 더 많은 파라미터보다 더 깊은 추론이 효과적일 수 있다는 발견은 Large Reasoning Models(LRM)이라는 새로운 모델 패밀리를 탄생시켰습니다.

## Large Reasoning Models (LRM)의 분류

2025년 현재 LRM은 AI 연구의 최전선입니다. 접근 방식에 따라 다음과 같이 분류할 수 있습니다.

### 분류 체계

**RL 기반 LRM**: RL로 추론 체인의 품질을 직접 최적화합니다.
- DeepSeek-R1: GRPO 기반, 4단계 파이프라인
- OpenAI o1/o3: 내부 RL 알고리즘 (비공개)
- Kimi k1.5: 장문 CoT + RL

**자기 학습 기반 LRM**: 자체 생성 데이터로 반복 학습합니다.
- STaR/Quiet-STaR: rationalization 기반 자기 학습
- ReST: 생성-필터링-학습 루프

**테스트 타임 계산 기반**: 추론 시 더 많은 계산을 투자합니다.
- Best-of-N + PRM: 여러 후보 중 검증기로 선택
- MCTS 기반: 트리 탐색으로 최적 추론 경로 탐색

| 모델 | 기관 | 방법 | 주요 특징 |
|-----|-----|-----|---------|
| o1/o3 | OpenAI | RL (비공개) | 최고 성능, 클로즈드 |
| DeepSeek-R1 | DeepSeek | GRPO+SFT | 오픈소스, 재현 가능 |
| Claude 3.7 Sonnet | Anthropic | RL (비공개) | Extended Thinking |
| Gemini 2.0 Flash Thinking | Google | RL (비공개) | 다모달 추론 |
| QwQ-32B | Alibaba | RL | 오픈소스, 32B |
| Kimi k1.5 | Moonshot | RL+장문 CoT | 컨텍스트 활용 |

이들은 공통적으로 추론 토큰(reasoning tokens)을 생성하고, RL로 추론 품질을 강화하며, 자기 검증을 수행합니다. 추론 강화는 단순한 프롬프팅 기법을 넘어, LLM 훈련의 핵심 축으로 자리잡았습니다.
