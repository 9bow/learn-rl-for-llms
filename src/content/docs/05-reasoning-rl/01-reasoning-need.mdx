## LLM의 추론 한계

대형 언어 모델(LLM)은 자연어 이해, 번역, 요약 등의 태스크에서 인간 수준을 달성했지만, 엄밀한 추론이 필요한 영역에서는 체계적인 실패 패턴을 보인다.

**수학 문제**: 단순 산술은 잘 처리하지만, 다단계 계산이나 대수적 조작에서 중간 계산 오류가 누적된다. 초등학교 수준의 GSM8K 벤치마크에서도 초기 GPT-3는 약 5%의 정확도에 불과했다.

**논리 퍼즐**: "모든 A는 B다, 모든 B는 C다, 따라서 모든 A는 C다"와 같은 삼단논법은 잘 처리하지만, 반사실적 조건(counterfactual), 부정 논리, 다층 중첩 조건에서 일관성을 유지하지 못한다.

**코딩 과제**: 간단한 함수 작성은 가능하지만, 복잡한 알고리즘 설계, 버그 추적, 엣지 케이스 처리에서 오류를 범한다. 특히 모델이 "자신 있게" 틀린 코드를 생성하는 과신(overconfidence) 문제가 심각하다.

이러한 실패의 근본 원인은 LLM이 다음 토큰을 예측하는 과정에서 **중간 추론 상태를 명시적으로 유지하지 않기** 때문이다. 모델은 입력에서 출력으로 직접 매핑하며, 복잡한 문제에 필요한 계획(planning)과 검증(verification) 단계가 없다.

## Chain-of-Thought 프롬프팅

**CoT(Chain-of-Thought)** 프롬프팅은 Wei et al. (2022)이 제안한 방법으로, 모델이 최종 답 이전에 중간 추론 단계를 텍스트로 생성하도록 유도한다. 예시 프롬프트를 포함한 few-shot 방식과, "Let's think step by step"이라는 단일 문구를 추가하는 zero-shot 방식이 있다.

CoT는 수학, 논리, 상식 추론 벤치마크에서 큰 성능 향상을 가져왔다. 특히 모델 크기가 커질수록 CoT의 효과가 두드러지는 **창발(emergent)** 현상이 관찰되었다. 100B 파라미터 이상의 모델에서야 CoT의 이점이 나타나기 시작했다.

그러나 CoT에는 한계가 있다. 첫째, 올바른 추론 경로를 보여주는 예시가 없으면 모델이 틀린 방향으로 추론할 수 있다. 둘째, 추론 단계 자체가 부정확할 수 있으며, 모델은 오류를 자기 수정하지 않는다. 셋째, 추론 깊이가 프롬프트 길이에 제한되어 매우 복잡한 문제에는 역부족이다.

## System 1 vs System 2 사고

심리학자 Daniel Kahneman의 이중 과정 이론에 따르면, 인간의 사고는 두 가지 시스템으로 구성된다.

- **System 1**: 빠르고, 자동적이며, 직관적인 사고. 노력 없이 즉각적으로 작동한다.
- **System 2**: 느리고, 의도적이며, 분석적인 사고. 복잡한 문제에 주의를 집중하여 단계별로 처리한다.

표준 LLM은 전형적인 **System 1 기계** 다. 단일 순전파로 토큰을 생성하며, 어려운 문제와 쉬운 문제에 동일한 계산을 사용한다. "생각의 시간"을 늘리는 것이 불가능하다.

추론 강화 연구의 핵심 목표는 LLM에 **System 2 능력을 부여** 하는 것이다. 이를 위해 추론 토큰을 생성하고, 자기 검증을 수행하며, 필요 시 백트래킹하는 능력이 요구된다.

## o1 모델과 패러다임 전환

OpenAI의 **o1 모델** (2024)은 이 분야의 패러다임을 바꾼 이정표다. o1은 추론 과정에서 긴 내부 "생각(thinking)" 체인을 생성하며, 이 과정에 RL을 적용하여 추론 능력을 강화했다. AIME 2024에서 GPT-4o의 13.4%에서 83.3%로, AMC 2023에서 96.7%로 성능이 도약했다.

o1의 핵심 통찰은 **추론 시간 계산(test-time compute)을 확장** 하면 성능이 향상된다는 것이다. 더 많은 파라미터보다 더 깊은 추론이 효과적일 수 있다는 발견은 Large Reasoning Models(LRM)이라는 새로운 모델 패밀리를 탄생시켰다.

## Large Reasoning Models (LRM)의 시대

2025년 현재 LRM은 AI 연구의 최전선이다. DeepSeek-R1, Claude 3.7 Sonnet (extended thinking), Gemini 2.0 Flash Thinking 등이 대표적이다. 이들은 공통적으로:

- **추론 토큰(reasoning tokens)**: 최종 답 이전에 긴 내부 추론 과정 생성
- **RL 기반 훈련**: 검증 가능한 보상으로 추론 품질 강화
- **자기 검증**: 생성된 추론 경로를 스스로 평가하고 수정

이 접근의 효과는 수학, 코딩, 과학 벤치마크에서 명확히 입증되었다. 추론 강화는 단순한 프롬프팅 기법을 넘어, LLM 훈련의 핵심 축으로 자리잡았다.
