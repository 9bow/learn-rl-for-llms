## 비교의 맥락

DPO와 PPO 기반 RLHF는 동일한 목표(인간 선호에 맞는 LLM 훈련)를 서로 다른 방식으로 달성합니다. 어느 쪽이 더 낫다는 단순한 답은 없으며, 태스크 유형, 데이터 규모, 컴퓨팅 자원에 따라 최적 선택이 달라진다.

## 성능 비교

**단순 과제에서는 DPO가 유리합니다.** 요약, 감성 조절, 단순 지시 따르기 등의 태스크에서 DPO는 PPO와 동등하거나 더 나은 결과를 보입니다. 구현이 간단하고 학습이 안정적이므로 실용적 우위가 있습니다.

**복잡한 과제에서는 PPO가 유리한 경향이 있습니다.** 수학 추론, 코드 생성, 다단계 논리 등 탐색이 중요한 영역에서는 PPO 기반 RLHF가 더 강력하다. DeepSeek-R1, OpenAI o1 등 최첨단 추론 모델들이 RL(GRPO/PPO)을 핵심으로 사용하는 것이 이를 뒷받침합니다.

**분포 이동 문제**: DPO는 고정된 오프라인 데이터셋에서 학습합니다. 학습이 진행될수록 정책 분포가 원래 데이터 수집 분포와 멀어지며, 이 경우 학습 신호의 대표성이 감소합니다. PPO는 온라인으로 새 샘플을 생성하므로 이 문제가 없습니다.

## 계산 비용 비교

| 항목 | DPO | PPO 기반 RLHF |
|------|-----|-------------|
| 필요 모델 수 | 2개 (정책 + 참조) | 4개 (정책 + 참조 + 보상 + 가치) |
| 학습 단계 | 1단계 | 2단계 이상 |
| 온라인 샘플링 | 불필요 | 필수 (배치마다) |
| 하이퍼파라미터 수 | 적음 ($\beta$ 등) | 많음 (클리핑, GAE 등) |
| 엔지니어링 복잡도 | 낮음 | 높음 |
| 훈련 시간 | 상대적으로 짧음 | 상대적으로 김 |

7B 모델 기준으로 DPO는 단일 A100 노드에서 수 시간 내 완료 가능한 반면, PPO는 분산 학습 환경과 더 긴 시간이 필요합니다.

## 안정성 비교

DPO의 학습은 지도 학습과 유사하여 매우 안정적입니다. 손실이 단조적으로 감소하고, 하이퍼파라미터에 대한 민감도가 낮습니다.

PPO는 강화학습 특유의 불안정성이 있습니다. 보상 해킹, KL 폭발, 값 함수 오추정 등의 문제가 발생할 수 있으며, 이를 제어하기 위한 클리핑, 엔트로피 보너스, 학습률 스케줄 등 세심한 엔지니어링이 필요합니다.

## 탐색 능력 비교

PPO의 핵심 강점은 **온라인 탐색(online exploration)** 입니다. 학습 중 모델이 새로운 응답을 생성하고, 보상 모델이 이를 평가하므로, 데이터에 없던 고품질 출력을 발견할 수 있습니다. 이는 특히 추론 능력 향상에 결정적입니다.

DPO는 오프라인 방식이므로 탐색 능력이 없습니다. 주어진 선호 쌍 데이터의 품질에 결과가 강하게 의존합니다.

## 하이브리드 접근법

두 방법의 단점을 보완하는 하이브리드 전략들이 연구되고 있습니다.

**Iterative DPO (Online DPO)**: 정책이 새 응답을 샘플링하고, 보상 모델 또는 AI 피드백으로 선호 쌍을 생성하여 반복적으로 DPO를 수행합니다. 오프라인 DPO의 분포 이동 문제를 완화합니다.

**RLHF Workflow (Dong et al., 2024)**: SFT → 보상 모델 훈련 → 반복적 온라인 RLHF의 체계적 파이프라인을 제시하며, DPO를 오프라인 초기화 단계로 사용하고 이후 온라인 PPO로 정제합니다.

**DPO → PPO 전이**: DPO로 초기 정렬을 수행한 후 PPO로 미세 조정하는 2단계 접근도 실용적으로 사용됩니다.

## 현재 컨센서스

| 차원 | 우위 | 근거 |
|------|-----|-----|
| 구현 용이성 | DPO | 단순한 손실 함수, 적은 모델 수 |
| 학습 안정성 | DPO | 지도 학습 수준의 안정성 |
| 복잡 추론 성능 | PPO | o1, R1 등 최첨단 모델의 선택 |
| 탐색 능력 | PPO | 온라인 샘플링 |
| 데이터 효율 | PPO | 온라인 학습으로 새 정보 활용 |
| 컴퓨팅 효율 | DPO | 적은 모델, 짧은 훈련 |

2025년 현재, 리소스가 제한된 환경이나 단순한 정렬 태스크에는 DPO, 최첨단 추론 성능이 필요한 경우에는 PPO 또는 GRPO 기반 RLVR이 선호되는 추세다. 두 방법은 경쟁 관계보다 상호 보완적인 도구로 이해하는 것이 실용적입니다.
