## DPO 암묵적 보상과의 수학적 비교

DPO의 암묵적 보상은 정책 비율(log-ratio)로 정의된다.

$$
r_{\text{DPO}}(x, y) = \beta \log \frac{\pi_\theta(y \mid x)}{\pi_{\text{ref}}(y \mid x)}
$$

이 보상은 두 가지 문제를 내포한다. 첫째, 참조 모델 $\pi_{\text{ref}}$가 반드시 필요하므로 학습 시 두 모델을 동시에 메모리에 올려야 한다. 둘째, $\log \pi_\theta(y|x) = \sum_{t=1}^{|y|} \log \pi_\theta(y_t|x, y_{<t})$는 토큰 수에 비례하는 합산값이므로, 길이가 다른 응답 간 직접 비교가 불공평하다.

SimPO는 참조 모델 의존성을 제거하고 길이를 명시적으로 정규화함으로써 두 문제를 모두 해결한다.

## SimPO의 핵심 아이디어

**SimPO(Simple Preference Optimization)** 는 Princeton NLP 그룹이 2024년 제안한 방법으로, DPO보다 단순하면서 더 강력한 성능을 보입니다. 핵심 아이디어는 두 가지다.

첫째, 참조 모델(reference model)을 완전히 제거합니다. DPO는 정책 비율 $\frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$를 암묵적 보상으로 사용하지만, SimPO는 **평균 로그 확률** 을 직접 암묵적 보상으로 사용합니다.

$$
r_{\text{SimPO}}(x, y) = \frac{1}{|y|} \log \pi_\theta(y \mid x) = \frac{1}{|y|} \sum_{t=1}^{|y|} \log \pi_\theta(y_t \mid x, y_{<t})
$$

둘째, 응답 길이 $|y|$로 나누어 **길이 정규화** 를 수행합니다. 이는 긴 응답이 단순히 더 높은 확률 합계를 가지는 길이 편향(length bias)을 방지합니다.

## SimPO 손실 함수

SimPO는 BT 모델 구조를 유지하면서 마진 항 $\gamma$를 추가합니다.

$$
L_{\text{SimPO}}(\pi_\theta) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}}\!\left[\log \sigma\!\left(\frac{\beta}{|y_w|}\log \pi_\theta(y_w|x) - \frac{\beta}{|y_l|}\log \pi_\theta(y_l|x) - \gamma\right)\right]
$$

여기서 $\gamma > 0$는 **목표 보상 마진** 으로, 선호 응답과 비선호 응답 사이에 최소 마진을 강제합니다. 이는 DPO에 없는 요소로, 경계 사례에서 더 명확한 분리를 유도합니다. 논문에서는 $\beta = 2.5$, $\gamma = 1.4$가 권장 기본값으로 사용되었다.

## 보상 마진 γ의 역할

DPO 손실은 선호 응답과 비선호 응답 사이에 어떠한 최소 마진도 강제하지 않는다. 두 보상이 같은 방향으로만 순서가 맞으면 손실이 0이 될 수 있다. 이는 경계 사례(margin이 거의 없는 경우)에서 분류 경계가 불분명해지는 문제를 유발한다.

SimPO의 $\gamma > 0$는 선호 응답의 보상이 비선호 응답보다 최소 $\gamma$만큼 커야 손실이 감소하도록 강제한다.

$$
\frac{\beta}{|y_w|}\log \pi_\theta(y_w|x) - \frac{\beta}{|y_l|}\log \pi_\theta(y_l|x) > \gamma
$$

이 마진은 두 가지 효과를 가진다. 첫째, 경계 사례에서 더 명확한 결정 경계를 학습하게 하여 일반화 성능을 향상시킨다. 둘째, 모델이 단순히 선호/비선호 방향만 맞추는 게 아니라 일정 수준 이상의 분리를 달성하도록 유도하여 응답 품질 격차를 키운다. 논문 실험에서 $\gamma = 0$으로 설정하면 성능이 유의하게 하락했으며, $\gamma = 1.4$ 전후가 최적으로 나타났다.

## 참조 모델 제거의 실용적 이점

참조 모델을 제거함으로써 얻는 실용적 이점은 상당하다.

| 항목 | DPO | SimPO |
|------|-----|-------|
| 학습 시 GPU 메모리 | 정책 + 참조 모델 | 정책 모델만 |
| 배치당 순전파 횟수 | 2회 (정책 + 참조) | 1회 |
| 구현 복잡도 | 참조 모델 관리 필요 | 단순 |
| 추론 시 추가 비용 | 없음 | 없음 |

7B 모델 기준으로 참조 모델 제거는 약 절반의 GPU 메모리를 절약하며, 배치 처리 속도도 향상됩니다. 대형 모델일수록 이 이점은 더욱 커진다.

## AlpacaEval 2 성능

SimPO는 AlpacaEval 2 벤치마크에서 DPO 대비 현저한 성능 향상을 보고했다. AlpacaEval 2는 GPT-4가 응답 품질을 평가하는 자동 평가 벤치마크로, 길이 제어 승률(Length-Controlled Win Rate, LC-WR)이 주요 지표다.

Llama-3-8B-Instruct 기반 실험에서:
- **DPO**: LC-WR 26.3%
- **SimPO**: LC-WR 32.7% (약 **+6.4점** 향상)

Llama-3-70B-Instruct 기반에서도 유사한 경향을 보였으며, MT-Bench, Arena-Hard 등 다른 벤치마크에서도 DPO보다 일관되게 높은 성능을 기록했다. 이 논문은 NeurIPS 2024에 발표되었다.

## 상세 벤치마크 결과

SimPO 논문(Meng et al., 2024, NeurIPS 2024)은 세 가지 주요 벤치마크에서 DPO 및 다른 방법들과 비교했다.

**AlpacaEval 2 (Length-Controlled Win Rate)**

| 기반 모델 | DPO | IPO | ORPO | **SimPO** |
|-----------|-----|-----|------|-----------|
| Llama-3-8B-Instruct | 26.3% | 23.3% | 24.9% | **32.7%** |
| Llama-3-70B-Instruct | 34.4% | 32.6% | 29.8% | **44.7%** |
| Mistral-7B-Instruct | 18.7% | 17.1% | 20.4% | **24.5%** |

**Arena-Hard**

| 기반 모델 | DPO | **SimPO** |
|-----------|-----|-----------|
| Llama-3-8B-Instruct | 33.8% | **41.6%** |
| Llama-3-70B-Instruct | 52.4% | **65.5%** |

**MT-Bench**

MT-Bench에서도 SimPO는 DPO 대비 일관된 향상을 보였으며, 특히 수학·추론 카테고리에서 차이가 두드러졌다.

이 결과들은 단순한 두 가지 변경(참조 모델 제거 + 마진 추가)만으로 폭넓은 모델 크기와 벤치마크에서 일관된 성능 향상이 가능함을 보여준다.

## DPO vs SimPO vs ORPO 비교

| 항목 | DPO | SimPO | ORPO |
|------|-----|-------|------|
| 참조 모델 필요 | 예 | **아니오** | 아니오 |
| 보상 마진 강제 | 아니오 | **예 (γ)** | 아니오 |
| 길이 정규화 | 아니오 | **예** | 아니오 |
| GPU 메모리 | 정책 + 참조 | **정책만** | 정책만 |
| 손실 구조 | 독립적 분류 | 독립적 분류 | SFT + 선호 결합 |
| 하이퍼파라미터 | β | β, γ | λ |
| AlpacaEval 2 (8B) | 26.3% | **32.7%** | 24.9% |

ORPO는 SFT 손실과 선호 손실을 결합하여 SFT 단계를 생략할 수 있지만, 마진 강제가 없어 경계 사례 처리가 약하다. SimPO는 참조 모델과 SFT 단계를 모두 유지하되 마진과 길이 정규화를 추가하는 전략을 택해 성능과 효율성을 동시에 달성한다.

## 길이 정규화의 중요성

길이 정규화 없이 단순히 $\log \pi_\theta(y|x)$를 보상으로 사용하면, 모델은 긴 응답을 선호하도록 편향됩니다. 자동 회귀 언어 모델의 확률은 토큰이 추가될수록 감소하는 경향이 있어, 짧은 응답이 수학적으로 더 높은 확률을 가질 수 있습니다. 길이 정규화는 이 편향을 제거하고, 실제 내용 품질에 기반한 선호 학습을 가능하게 합니다.

## 정리

SimPO는 "단순할수록 좋다"는 원칙의 좋은 예다. 참조 모델을 제거하고 평균 로그 확률과 마진을 더하는 최소한의 변경만으로 DPO를 능가하는 성능을 달성했다. 이는 DPO의 참조 모델이 실제로 필요하지 않을 수 있다는 점을 시사하며, 이후 ORPO, DAPO 등 참조 모델 없는 방법들의 흐름과도 맥을 같이 합니다.
