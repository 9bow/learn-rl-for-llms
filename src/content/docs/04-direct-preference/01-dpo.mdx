## 왜 DPO가 등장했는가

RLHF(Reinforcement Learning from Human Feedback) 파이프라인은 인간 선호 데이터로 보상 모델을 훈련하고, 이 보상 모델을 이용해 PPO 같은 RL 알고리즘으로 언어 모델을 최적화하는 2단계 구조입니다. 이 방식은 강력하지만 실용적인 문제들이 많습니다. 보상 모델과 정책 모델, 참조 모델, 가치 함수(critic) 등 최대 4개의 모델을 동시에 메모리에 올려야 하며, PPO 하이퍼파라미터 튜닝이 매우 민감하고, 보상 해킹(reward hacking) 위험도 상존합니다.

Rafailov et al. (2023)은 이러한 복잡성을 근본적으로 해소하는 **DPO(Direct Preference Optimization)** 를 제안했다. 핵심 통찰은 다음과 같다: KL-정규화 보상 최대화 목표 하에서 최적 정책은 closed-form으로 존재하며, 이를 이용해 보상 함수를 정책 비율(policy ratio)로 재표현할 수 있습니다. 결과적으로 보상 모델을 별도로 학습하지 않고, 선호 쌍 데이터만으로 정책을 직접 최적화할 수 있습니다.

## DPO 손실 함수

DPO의 학습 목표는 다음 손실 함수를 최소화하는 것입니다.

$$
L_{\text{DPO}}(\pi_\theta; \pi_{\text{ref}}) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)}\right)\right]
$$

여기서:
- $x$ : 입력 프롬프트
- $y_w$ : 선호 응답 (winner)
- $y_l$ : 비선호 응답 (loser)
- $\pi_\theta$ : 학습 중인 정책 (LM)
- $\pi_{\text{ref}}$ : 고정된 참조 정책 (SFT 모델)
- $\beta$ : KL 페널티 강도를 조절하는 온도 하이퍼파라미터
- $\sigma$ : 시그모이드 함수

손실 함수를 직관적으로 해석하면, 시그모이드 내부 항은 **선호 응답에 대한 암묵적 보상 마진** 입니다. $\pi_\theta$가 참조 정책 대비 $y_w$를 더 선호하고 $y_l$을 덜 선호할수록 이 값이 커지고 손실이 줄어든다. 즉, 모델이 올바른 방향으로 선호를 학습하면 학습이 자연스럽게 수렴합니다.

## RLHF 대비 구현 단순성

| 항목 | RLHF (PPO) | DPO |
|------|-----------|-----|
| 필요 모델 수 | 4개 (정책, 참조, 보상, 가치) | 2개 (정책, 참조) |
| 학습 단계 | 2단계 (보상 모델 → RL) | 1단계 |
| 알고리즘 복잡도 | PPO (고) | 지도 학습 수준 (저) |
| 보상 해킹 위험 | 있음 | 없음 (암묵적 보상) |
| 하이퍼파라미터 민감도 | 높음 | 낮음 |
| GPU 메모리 | 대용량 필요 | 상대적으로 효율적 |

DPO는 구현 관점에서 단순한 binary cross-entropy 분류 손실과 동일한 구조를 가집니다. 표준 딥러닝 훈련 루프에 그대로 통합 가능하며, Hugging Face `trl` 라이브러리에서도 `DPOTrainer`로 지원됩니다.

## 실전 적용 사례

DPO는 발표 직후 LLM 미세조정 연구에서 폭넓게 채택되었다. 대표적인 적용 사례는 다음과 같습니다.

**Zephyr-7B** (Tunstall et al., 2023): Mistral-7B를 기반으로 DPO를 적용하여 GPT-3.5-turbo 수준의 성능을 7B 모델에서 달성했다. AI Feedback(AIF) 데이터를 활용한 distilled DPO가 핵심이었다.

**Llama 계열 미세조정**: Meta의 Llama 2/3 모델들도 DPO 변형을 통해 Helpful/Harmless 성능을 강화했으며, 다양한 오픈소스 커뮤니티 모델들이 DPO 기반으로 훈련되고 있습니다.

**실전 주의점**: DPO는 오프라인(offline) 방식으로 고정된 데이터셋에서 학습합니다. 이는 분포 이동(distribution shift) 문제를 야기할 수 있는데, 학습 중 생성되는 정책 분포가 참조 분포와 멀어질수록 선호 쌍의 대표성이 떨어진다. 이를 해결하기 위한 Iterative DPO, Online DPO 등의 변형이 연구되고 있습니다. 또한 $\beta$ 값의 선택이 성능에 민감하게 작용하며, 일반적으로 0.01에서 0.5 사이에서 태스크에 맞게 조정합니다.

## 정리

DPO는 RLHF의 복잡성을 획기적으로 낮추면서도 경쟁력 있는 선호 정렬 성능을 달성합니다. 보상 모델 없이 선호 쌍 데이터만으로 언어 모델을 직접 최적화한다는 아이디어는 이후 수많은 변형 알고리즘(IPO, KTO, SimPO 등)의 토대가 되었으며, 현재 LLM 정렬 연구의 핵심 패러다임으로 자리잡았다.
