## On-policy vs Off-policy: 핵심 차이

강화학습 알고리즘은 데이터를 어떻게 수집하고 활용하는지에 따라 크게 두 범주로 나뉩니다.

**On-policy(온-폴리시)** 학습은 현재 학습 중인 정책 $\pi_\theta$ 자체로 데이터를 수집하고, 그 데이터로 즉시 업데이트합니다. 업데이트 후에는 이전 데이터를 폐기합니다.

**Off-policy(오프-폴리시)** 학습은 행동 정책(behavior policy) $\mu$로 수집한 데이터를 목표 정책(target policy) $\pi_\theta$ 학습에 재사용할 수 있습니다. 리플레이 버퍼(replay buffer)에 과거 경험을 저장하여 반복 활용합니다.

| 특성 | On-policy | Off-policy |
|------|-----------|------------|
| 대표 알고리즘 | REINFORCE, PPO, A3C | Q-learning, DQN, SAC |
| 데이터 수집 | 현재 정책으로 직접 | 별도 행동 정책 또는 기존 데이터셋 |
| 샘플 효율성 | 낮음 (데이터 재사용 불가) | 높음 (리플레이 버퍼 활용) |
| 분산 | 낮음 | 높음 (중요도 샘플링 필요) |
| 구현 복잡도 | 상대적으로 단순 | 복잡 |

## 오프라인 RL의 정의와 특수성

**오프라인 RL(Offline RL)** 은 사전에 수집된 고정 데이터셋 $\mathcal{D} = \{(s, a, r, s')\}$만으로 학습하며, 환경과의 추가 상호작용이 전혀 없습니다. 이는 의료, 자율주행처럼 실시간 탐색이 위험하거나 비용이 매우 큰 도메인에 적합합니다.

LLM 맥락에서 오프라인 RL은 사전 수집된 선호 데이터셋 $\mathcal{D} = \{(x, y_w, y_l)\}$ (프롬프트, 선호 응답, 비선호 응답)으로만 학습하는 DPO, RAFT, SLiC 등의 접근법을 포함합니다.

## LLM에서의 온라인 RLHF: PPO 기반

PPO 기반 RLHF는 전형적인 온라인 접근법입니다:

1. 현재 정책 $\pi_\theta$로 프롬프트에 대한 응답을 샘플링 (롤아웃)
2. 보상 모델 $r_\phi$로 각 응답에 점수 부여
3. PPO로 정책 업데이트
4. 반복

**온라인 RLHF의 장점:**
- 현재 정책의 분포에서 데이터를 수집하므로 분포 이동 문제 없음
- 탐색(exploration)을 통해 새로운 고보상 반응 발견 가능
- 정책이 개선될수록 더 좋은 훈련 신호를 자연스럽게 획득

**온라인 RLHF의 단점:**
- 4개 모델(Actor, Critic, Reward, Reference) 동시 메모리 유지 필요
- 학습 불안정성 (Section 02-05에서 상세 다룸)
- 보상 모델 추론 비용이 훈련 비용에 추가
- 롤아웃 생성이 병목(bottleneck)이 될 수 있음

## LLM에서의 오프라인 RL: DPO 기반

DPO(Rafailov et al., 2023)는 KL 제약 RL의 최적해를 닫힌 형태로 표현하여 보상 모델 없이 직접 정책을 학습합니다:

$$\mathcal{L}_{DPO}(\pi_\theta; \pi_{ref}) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{ref}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{ref}(y_l \mid x)} \right) \right]$$

선호 응답 $y_w$의 상대적 로그 확률을 높이고, 비선호 응답 $y_l$의 상대적 로그 확률을 낮추는 방향으로 학습합니다.

**오프라인 RL(DPO)의 장점:**
- 보상 모델 별도 학습 불필요 (2개 모델로 충분)
- 구현 단순성: SFT와 유사한 학습 루프
- 안정적인 학습 (PPO 특유의 불안정성 없음)
- 계산 비용이 낮음

**오프라인 RL(DPO)의 단점:**
- **분포 이동(Distribution Shift)**: 학습 데이터가 특정 정책으로 생성된 경우, 정책이 발전하면 데이터가 outdated됨
- **탐색 부재**: 데이터셋에 없는 새로운 고보상 반응을 발견할 수 없음
- **데이터 품질 의존성**: 기존 선호 데이터의 질과 다양성에 크게 의존

## 분포 이동 문제의 심각성

오프라인 RL에서 분포 이동은 치명적입니다. 학습 데이터가 정책 $\pi_{old}$로 수집되었는데, 현재 정책 $\pi_\theta$가 크게 달라지면 학습 신호가 부정확해집니다.

Ramamurthy et al. (2023)은 NLP 과제에서 온라인 vs 오프라인 RL을 체계적으로 비교하여, 온라인 방법이 장기적으로 오프라인 방법보다 우수한 성능을 보임을 입증했습니다. 특히 탐색이 중요한 과제에서 그 차이가 두드러졌습니다.

## 하이브리드 접근법

실제 최신 연구들은 온라인과 오프라인의 장점을 결합하는 방향으로 발전하고 있습니다:

- **Iterative DPO**: 현재 정책으로 새 데이터를 생성하고 DPO로 반복 업데이트
- **Online DPO**: 실시간으로 정책의 응답을 샘플링하여 선호 비교 후 DPO 적용
- **RAFT(Reward rAnked Fine-Tuning)**: 보상 모델로 점수를 매겨 상위 응답만 SFT

| 방법 | 온라인/오프라인 | 보상 모델 필요 | 계산 비용 |
|------|----------------|--------------|----------|
| PPO-RLHF | 온라인 | 필요 | 매우 높음 |
| DPO | 오프라인 | 불필요 | 낮음 |
| Iterative DPO | 하이브리드 | 불필요 | 중간 |
| RAFT | 하이브리드 | 필요 | 중간 |
| Online DPO | 온라인 | 불필요 | 중간 |

연구 트렌드는 온라인 데이터 수집의 탐색 이점을 유지하면서 DPO의 구현 단순성을 결합하는 방향으로 이동하고 있습니다.
