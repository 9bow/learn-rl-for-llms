## Best-of-N 샘플링 개요

**Best-of-N(BoN) 샘플링** 은 훈련 없이 추론 시점에서만 모델의 성능을 향상시키는 가장 단순하고 강력한 기법입니다. 절차는 다음과 같습니다:

1. 동일한 프롬프트에 대해 $N$개의 응답을 독립적으로 생성합니다
2. 보상 모델(Reward Model) 또는 검증기가 각 응답에 점수를 부여합니다
3. 점수가 가장 높은 응답을 최종 출력으로 선택합니다

$$\hat{y} = \arg\max_{y_i \in \{y_1, \ldots, y_N\}} r(x, y_i)$$

여기서 $r(x, y_i)$는 보상 모델이 프롬프트 $x$에 대한 응답 $y_i$에 부여하는 점수입니다.

## 이론적 기반

### BoN의 성능 이득

$N$개의 샘플 중 최고를 선택했을 때 기대 품질은 이론적으로 분석 가능합니다. 보상 $r$이 정규분포를 따른다고 가정하면, $N$번째 순서 통계량(order statistic)의 기댓값은:

$$\mathbb{E}[\max_N r] \approx \mu + \sigma \cdot \sqrt{2 \ln N}$$

이는 $N$이 증가할수록 **로그 스케일** 로 성능이 향상됨을 의미합니다. 즉, 처음에는 $N$을 늘리는 것이 효과적이지만, 수익이 체감(diminishing returns)합니다.

### 보상 모델 정렬과 실제 품질

이론적 분석은 보상 모델 점수와 실제 품질이 완벽히 일치한다고 가정합니다. 실제로는 **보상 모델은 프록시(proxy)** 이며, 완벽한 품질 척도가 아닙니다.

BoN이 실제로 동작하는 이유: 언어 모델의 디코딩 분포는 실제 선호 분포보다 더 넓습니다. 모델은 "좋은 응답"을 생성할 능력을 갖추고 있지만, 매번 그것을 선택하지는 못합니다. BoN은 이 '잠재적 능력'을 추론 계산으로 끌어냅니다.

## 보상 해킹 리스크

BoN은 $N$이 충분히 크면 보상 해킹(reward hacking)이 발생합니다. 보상 모델의 허점을 찌르는 응답이 상위에 오를 수 있기 때문입니다.

예를 들어, 긴 응답이 보상 모델에서 체계적으로 높은 점수를 받는다면, 대규모 $N$ 샘플링에서는 불필요하게 길지만 점수가 높은 응답이 선택될 수 있습니다.

**보상 해킹의 수학적 이해**: 보상 모델 $r$이 실제 품질 $r^*$에 노이즈 $\epsilon$이 추가된 것이라면:

$$r(x, y) = r^*(x, y) + \epsilon(x, y)$$

$N$이 커질수록 $\max_N \epsilon$이 커져, 실제 품질과 무관한 노이즈에 의해 선택이 이루어질 수 있습니다.

## MBR-BoN: 근접성 정규화

**MBR-BoN(Minimum Bayes Risk Best-of-N, arXiv:2404.01054)** 은 보상 해킹을 완화하기 위해 **최소 베이즈 위험(Minimum Bayes Risk)** 원리를 BoN에 결합합니다.

### 핵심 아이디어

단순히 보상 점수가 가장 높은 응답을 선택하는 대신, **다른 샘플들과 얼마나 근접한지** 도 함께 고려합니다:

$$\hat{y} = \arg\max_{y_i} \left[ \alpha \cdot r(x, y_i) + (1 - \alpha) \cdot \frac{1}{N} \sum_{j=1}^{N} \text{sim}(y_i, y_j) \right]$$

여기서 $\text{sim}(y_i, y_j)$는 두 응답 사이의 유사도(예: BLEU, BERTScore, 임베딩 코사인 유사도)이고, $\alpha$는 보상 점수와 근접성의 가중치를 조절합니다.

### 왜 효과적인가

보상 해킹 응답은 대개 다른 응답들과 크게 다릅니다. 비정상적으로 길거나, 특이한 패턴을 가진 응답은 군집의 중심에서 멀리 떨어집니다. MBR-BoN은 이런 이상치를 자연스럽게 걸러냅니다.

반면, 진정으로 좋은 응답은 보상 점수도 높으면서 다른 고품질 응답들과 유사한 경향이 있어 MBR 점수에서도 높은 순위를 유지합니다.

## SWIFT: 내재적 보상 마이닝

**SWIFT(arXiv:2505.12225)** 는 별도의 보상 모델 없이 LLM 자체의 **은닉 상태(hidden states)** 에서 내재적 보상 신호를 추출합니다.

### 핵심 발상

언어 모델은 자신의 응답 품질에 대한 암묵적 지식을 내부 표현에 담고 있습니다. 고품질 응답을 생성할 때와 저품질 응답을 생성할 때 은닉 상태의 패턴이 다릅니다. SWIFT는 이 차이를 학습하여 품질 신호로 활용합니다.

### 구조

SWIFT는 LLM의 은닉 상태 위에 **매우 작은 프로브(probe)** 를 추가합니다:

$$r_{SWIFT}(y) = f_\theta(h_{LLM}(y))$$

여기서 $h_{LLM}(y)$는 응답 $y$를 생성할 때의 LLM 은닉 상태이고, $f_\theta$는 학습 가능한 소형 네트워크입니다. 이 프로브는 **전체 파라미터의 0.005%** 에 해당합니다.

### 성능

SWIFT 기반 BoN은 MATH 데이터셋에서 **12.7%의 정확도 향상** 을 달성했으며, 별도로 훈련된 대형 보상 모델인 EurusRM-7B와 동등한 성능을 보였습니다. 이는 외부 보상 모델 없이 추론 품질을 향상시킬 수 있음을 보여줍니다.

## BoN 변형 비교

| 방법 | 보상 신호 | 해킹 완화 | 추가 비용 | 적합한 상황 |
|------|----------|----------|----------|------------|
| **표준 BoN** | 외부 보상 모델 | 없음 | 보상 모델 추론 | 신뢰할 수 있는 보상 모델 보유 |
| **MBR-BoN** | 외부 보상 모델 + 유사도 | 있음 (중간) | 쌍 유사도 계산 | 보상 해킹 우려 시 |
| **SWIFT-BoN** | LLM 은닉 상태 | 있음 (내재적) | 미미 (0.005%) | 별도 보상 모델 없는 경우 |
| **Verifier-BoN** | 규칙 기반 검증기 | 있음 (강함) | 검증기 실행 | 수학/코드 등 검증 가능 도메인 |

## 비용-성능 트레이드오프

BoN의 핵심 제약은 **계산 비용의 선형 증가** 입니다. $N$개의 응답을 생성하면 추론 비용도 $N$배가 됩니다.

성능 향상 대비 비용을 분석하면:

- $N = 4$: 비용 4배, 성능 $\approx \mu + 1.0\sigma$ 향상
- $N = 16$: 비용 16배, 성능 $\approx \mu + 1.5\sigma$ 향상
- $N = 64$: 비용 64배, 성능 $\approx \mu + 2.0\sigma$ 향상
- $N = 256$: 비용 256배, 성능 $\approx \mu + 2.4\sigma$ 향상

체감 수익 감소가 명확합니다. 실제 응용에서 $N = 8 \sim 32$ 범위가 비용 대비 효과적입니다.

### BoN vs. 추가 훈련

같은 예산으로 BoN 추론에 쓸 것인가, 아니면 모델을 더 훈련할 것인가? 연구에 따르면:

- **단기적**: 훈련된 모델에 BoN을 적용하는 것이 가장 효율적
- **중기적**: BoN으로 수집된 선호 데이터로 추가 DPO/PPO 훈련
- **장기적**: 훈련으로 모델 자체를 개선하는 것이 BoN보다 계산 효율적

이상적인 전략은 BoN으로 빠르게 성능을 확인하고, 안정적으로 효과가 있는 경우에만 훈련을 통해 그 능력을 내재화하는 것입니다.
