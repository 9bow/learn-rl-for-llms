## PPO-RLHF 학습 루프 개요

PPO-RLHF는 다음 세 단계의 반복 루프로 구성됩니다:

1. **롤아웃(Rollout) 생성**: Actor 정책으로 응답 샘플링
2. **보상 계산(Reward Computation)**: 보상 모델과 KL 페널티로 토큰별 보상 계산
3. **PPO 업데이트(PPO Update)**: Advantage 추정 후 Actor와 Critic 업데이트

이 루프는 수천~수만 번 반복되며, 각 반복을 **PPO 이터레이션(iteration)** 이라 합니다.

## 4개 모델의 역할

PPO-RLHF는 동시에 4개의 모델을 유지합니다:

| 모델 | 역할 | 학습 여부 | 초기화 |
|------|------|----------|--------|
| **Actor** $\pi_\theta$ | 응답 생성, 정책 업데이트 대상 | 학습 | SFT 모델 |
| **Critic** $V_\phi$ | 상태 가치 추정, Advantage 계산용 | 학습 | 보상 모델 또는 SFT |
| **Reward Model** $r_\psi$ | 응답 품질 점수 부여 | 고정 | RM 학습 결과 |
| **Reference** $\pi_{ref}$ | KL 페널티 계산용 기준점 | 고정 | SFT 모델 |

모든 모델이 동일한 크기의 LLM이라면, 70B 파라미터 기준으로 각 모델이 bf16으로 약 140GB를 차지하여 전체 약 560GB의 GPU 메모리가 필요합니다. 실제로는 Critic을 LoRA로 학습하거나, Reference와 Reward를 CPU 오프로딩하는 등의 최적화를 적용합니다.

## 단계 1: 롤아웃 생성

프롬프트 배치 $\{x_i\}_{i=1}^{B}$를 샘플링한 후, Actor [정책](../../01-rl-fundamentals/00-notation/#policies)으로 응답을 생성합니다:

$$y_i \sim \pi_\theta(\cdot \mid x_i), \quad i = 1, \ldots, B$$

> **직관.** 각 프롬프트에 대해 현재 정책에서 **샘플**을 뽑아 경험 데이터를 만든다. 학습은 바로 이 분포 위에서 진행되므로, 샘플링 온도·길이 제약·배치 다양성이 PPO의 편향-분산을 직접 결정하는 역할을 한다. 같은 프롬프트에 여러 응답을 뽑으면 동일 $x$ 내 분산이 줄어 baseline 효과를 갖는다.

이때 각 토큰의 로그 확률 $\log \pi_\theta(y_{i,t} \mid x_i, y_{i,<t})$를 저장합니다 (이후 PPO 비율 계산에 사용). 동시에 Reference 모델의 로그 확률도 계산하여 저장합니다.

**실용적 고려사항:**
- 온도(temperature) $\tau \in [0.7, 1.0]$으로 샘플링하여 다양성 확보
- 최소/최대 길이 제약으로 매우 짧거나 긴 응답 방지
- 여러 응답을 동일 프롬프트에서 샘플링하면 분산 감소 (Group 기반 PPO)

## 단계 2: 보상 계산

각 토큰 위치 $t$에서의 [보상](../../01-rl-fundamentals/00-notation/#rewards-and-returns)을 계산합니다. 여기서 [\\(\beta\\)](../../01-rl-fundamentals/00-notation/#dpo)는 KL 페널티 계수입니다:

$$r_t = \begin{cases}
r_\psi(x_i, y_i) - \beta \log \frac{\pi_\theta(y_{i,T} \mid x_i, y_{i,<T})}{\pi_{ref}(y_{i,T} \mid x_i, y_{i,<T})} & t = T \text{ (마지막 토큰)} \\
-\beta \log \frac{\pi_\theta(y_{i,t} \mid x_i, y_{i,<t})}{\pi_{ref}(y_{i,t} \mid x_i, y_{i,<t})} & t < T
\end{cases}$$

> **직관.** 보상 모델의 점수 $r_\psi$는 문장이 끝난 뒤에만 한 번 주어지지만, KL 페널티는 매 토큰마다 로그비 형태로 **분배**되어 흐른다. 이로써 참조 정책에서 벗어나는 토큰이 발생 즉시 음의 신호를 받아 교정되고, 보상 해킹이 특정 구간에 몰리는 현상을 줄이는 역할을 한다.

그 다음 GAE(Generalized Advantage Estimation)로 [Advantage \\(A_t\\)](../../01-rl-fundamentals/00-notation/#value-functions)를 추정합니다:

$$\hat{A}_t = \sum_{l=0}^{T-t} (\gamma \lambda)^l \delta_{t+l}, \quad \delta_t = r_t + \gamma V_\phi(s_{t+1}) - V_\phi(s_t)$$

> **직관.** TD 오차 $\delta_t$를 $\gamma\lambda$로 지수 할인해 더해 Advantage를 만든다. $\lambda$가 0에 가까우면 편향이 큰 1-step TD에 가깝고, 1에 가까우면 분산이 큰 Monte Carlo에 가까워진다. 즉 $\lambda$는 편향과 분산의 저울을 결정짓는 손잡이 역할을 한다.

## 단계 3: PPO 업데이트

롤아웃 데이터 $\{(s_t, a_t, \hat{A}_t, V_t^{target})\}$로 여러 에포크에 걸쳐 미니배치 업데이트를 수행합니다.

**Actor 손실** (PPO-Clip, 비율 [\\(r_t(\theta)\\)](../../01-rl-fundamentals/00-notation/#ppo), 클립 폭 [\\(\epsilon\\)](../../01-rl-fundamentals/00-notation/#ppo)):

$$L^{Actor}(\theta) = -\mathbb{E}_t \left[ \min\left( r_t(\theta) \hat{A}_t,\ \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$

> **직관.** 비율 $r_t(\theta)$는 이전 정책 대비 현재 정책이 이 토큰을 **얼마나 더/덜** 선택하려는지를 잰다. $1-\epsilon \sim 1+\epsilon$ 바깥으로 벗어나면 그래디언트를 잘라내 이전 정책에서 과도하게 벗어나는 업데이트를 자동 방지하는 안전장치 역할을 한다. 즉 클립은 "한 스텝에 얼마까지만 믿고 움직일지"를 결정한다.

여기서 $r_t(\theta) = \frac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{old}}(a_t \mid s_t)}$는 현재 정책과 롤아웃 당시 정책의 비율입니다.

**Critic 손실** (가치 함수 학습):

$$L^{Critic}(\phi) = \mathbb{E}_t \left[ \left( V_\phi(s_t) - V_t^{target} \right)^2 \right]$$

> **직관.** Critic은 상태별 기대 수익을 회귀로 맞춘다. 타깃이 정확해질수록 Advantage의 분산이 줄어 Actor 업데이트가 안정되므로, Critic의 역할은 "Actor가 노이즈 아닌 신호를 보고 움직이게 하는 것"이다. Critic이 뒤처지면 GAE가 왜곡돼 학습이 비효율적으로 바뀐다.

$V_t^{target} = \hat{A}_t + V_\phi(s_t)$로 계산합니다.

## 핵심 하이퍼파라미터

| 하이퍼파라미터 | 기호 | 권장 범위 | 역할 |
|--------------|------|----------|------|
| Actor 학습률 | $\eta_{actor}$ | $10^{-6} \sim 10^{-5}$ | 정책 업데이트 보폭 |
| Critic 학습률 | $\eta_{critic}$ | $10^{-5} \sim 10^{-4}$ | 가치 함수 학습 속도 |
| PPO 클립 | $\epsilon$ | $0.1 \sim 0.2$ | 정책 변화 제한 |
| KL 계수 | $\beta$ | $0.1 \sim 0.5$ | 레퍼런스 모델 유지 |
| GAE $\lambda$ | $\lambda$ | $0.9 \sim 0.99$ | 편향-분산 트레이드오프 |
| 할인 인자 | $\gamma$ | $0.99 \sim 1.0$ | 미래 보상 가중치 |
| PPO 에포크 수 | $K$ | $1 \sim 4$ | 데이터 재사용 횟수 |
| 롤아웃 배치 크기 | $B$ | $64 \sim 512$ | 롤아웃 다양성 |

Hu et al. (2023)은 PPO 에포크 수 $K$가 너무 크면 (>4) 동일 데이터를 과도하게 재사용하여 on-policy 특성이 훼손되고 학습이 불안정해짐을 보고했습니다.

## 구현 팁: TRL 라이브러리

Hugging Face의 TRL(Transformer Reinforcement Learning) 라이브러리는 PPO-RLHF의 표준 구현을 제공합니다. 핵심 클래스는 `PPOTrainer`이며, 위의 3단계 루프를 자동으로 처리합니다.

**주요 안정화 기법 (Hu et al., 2023 기반):**

1. **Critic 초기화**: 보상 모델 가중치로 초기화하면 초기 Advantage 추정의 정확도가 높아짐
2. **보상 클리핑**: 극단적 보상값을 $[-5, 5]$ 등으로 클리핑하여 Advantage 폭발 방지
3. **Value Function 클리핑**: Critic 업데이트도 PPO 방식으로 클리핑 적용 가능
4. **적응형 KL**: 목표 KL 발산을 설정하고 $\beta$를 동적으로 조정 (OpenAI 방식)

적응형 [KL](../../01-rl-fundamentals/00-notation/#divergences) 제어:
$$\beta \leftarrow \begin{cases} \beta / 2 & \text{if } D_{KL} < D_{KL}^{target} / 1.5 \\ \beta \times 2 & \text{if } D_{KL} > D_{KL}^{target} \times 1.5 \\ \beta & \text{otherwise} \end{cases}$$

> **직관.** KL이 목표보다 작으면 페널티를 **절반**으로 줄여 더 적극적으로 움직이게 하고, 크면 **두 배**로 올려 레퍼런스 쪽으로 되돌린다. 즉 β를 수동으로 정하는 대신 KL을 목표 범위에 자동 고정하는 피드백 제어로, 학습 내내 "업데이트의 과감함"을 동적으로 결정하는 역할을 한다.

이 메커니즘은 KL 발산을 목표 범위 안에 자동으로 유지하여, $\beta$를 수동으로 튜닝하는 부담을 줄여줍니다.
