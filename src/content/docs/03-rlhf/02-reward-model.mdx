## 보상 모델의 역할

RLHF에서 보상 모델(Reward Model, RM)은 인간 선호의 **대리자(proxy)** 입니다. 수천 명의 인간 주석자가 실시간으로 모델 응답을 평가하는 것은 불가능하므로, 사전에 수집한 선호 데이터로 인간 판단을 모방하는 신경망을 학습합니다.

보상 모델은 프롬프트 $x$와 응답 $y$를 입력받아 해당 응답의 품질을 나타내는 스칼라 점수를 출력합니다:

$$r_\phi: \mathcal{X} \times \mathcal{Y} \rightarrow \mathbb{R}$$

**핵심 직관**: 보상 모델은 "이 응답이 인간에게 얼마나 만족스러울까?"를 수치화하는 역할을 합니다. 이 수치가 PPO 학습의 유일한 학습 신호이므로, 보상 모델의 정확도가 전체 RLHF 파이프라인의 품질 상한을 결정합니다.

## Bradley-Terry 모델

선호 데이터에서 보상 모델을 학습하는 가장 널리 사용되는 이론적 틀은 **Bradley-Terry 모델(1952)** 입니다. 이 모델은 두 선택지 $y_1, y_2$ 중 $y_1$이 선호될 확률을 다음과 같이 정의합니다:

$$P(y_1 \succ y_2 \mid x) = \sigma(r(x, y_1) - r(x, y_2)) = \frac{e^{r(x, y_1)}}{e^{r(x, y_1)} + e^{r(x, y_2)}}$$

여기서 $\sigma$는 시그모이드 함수, $r(x, y)$는 응답 $y$의 절대적 품질 점수입니다. 이 모델은 선호가 두 응답의 품질 차이에 의해서만 결정된다고 가정합니다.

**학습 목적함수 유도**: 관측된 선호 데이터 $\mathcal{D} = \{(x, y_w, y_l)\}$의 로그 가능도를 최대화하면,

$$\log P(\mathcal{D}) = \sum_{(x, y_w, y_l) \in \mathcal{D}} \log P(y_w \succ y_l \mid x) = \sum \log \sigma\bigl(r_\phi(x, y_w) - r_\phi(x, y_l)\bigr)$$

이를 최소화 문제로 바꾸면 **보상 모델 학습 손실**을 얻습니다:

$$\mathcal{L}_{RM}(\phi) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right) \right]$$

여기서 $y_w$는 선호(preferred/winner) 응답, $y_l$은 비선호(less preferred/loser) 응답입니다. 이 손실함수는 보상 모델이 선호 응답에 더 높은 점수를 부여하도록 학습합니다.

```python
# 보상 모델 학습 의사코드
def reward_model_loss(reward_model, batch):
    prompts, y_w, y_l = batch
    
    # 각 응답에 대한 보상 점수 계산
    r_w = reward_model(prompts, y_w)  # 선호 응답 점수 (스칼라)
    r_l = reward_model(prompts, y_l)  # 비선호 응답 점수 (스칼라)
    
    # Bradley-Terry 손실: 선호 응답 점수가 더 높아지도록
    loss = -F.logsigmoid(r_w - r_l).mean()
    return loss

for epoch in range(num_epochs):
    for batch in preference_dataloader:
        loss = reward_model_loss(reward_model, batch)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

## 선호 데이터 수집

선호 데이터는 동일 프롬프트에 대한 여러 응답을 주석자에게 보여주고 상대적 품질을 판단하게 하여 수집합니다.

**수집 방법의 유형:**

| 방법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 쌍별 비교(Pairwise) | 응답 A vs B 중 선호 선택 | 단순, 일관성 높음 | $O(n^2)$ 비교 필요 |
| 순위 매기기(Ranking) | $k$개 응답 전체 순위 | 정보량 많음 | 주석자 부담 큼 |
| 절대 평가(Likert) | 1~7점 절대 점수 | 직관적 | 주석자 간 기준 차이 |
| 최선/최악 선택 | 가장 좋은/나쁜 응답 선택 | 효율적 | 중간 품질 정보 손실 |

InstructGPT는 쌍별 비교와 순위 매기기를 조합하여 사용했습니다. 하나의 프롬프트에 대해 4~9개의 응답을 생성하고, 모든 쌍의 비교 정보를 추출하여 데이터 효율을 높였습니다. $k$개 응답에서 $\binom{k}{2}$개의 비교 쌍을 추출하므로, 9개 응답이면 36개의 학습 쌍을 얻습니다.

## 보상 모델 아키텍처

**기본 구조**: SFT 모델과 동일한 트랜스포머 구조에 스칼라 출력을 위한 선형 레이어(linear head)를 추가합니다.

$$r_\phi(x, y) = \mathbf{W} \cdot h_{[EOS]} + b$$

여기서 $h_{[EOS]}$는 EOS 토큰 위치의 마지막 레이어 은닉 상태, $\mathbf{W} \in \mathbb{R}^{1 \times d}$는 스칼라 출력을 위한 선형 변환입니다. 전체 시퀀스 $(x, y)$를 인코딩한 후 하나의 스칼라로 집약합니다.

**초기화**: 일반적으로 SFT 모델 가중치로 초기화합니다. 이는 (a) 학습 효율을 높이고, (b) 사전학습 언어 이해 능력을 보상 판단에 활용할 수 있게 합니다. 무작위 초기화 대비 수렴 속도가 빠르고 최종 성능이 높습니다.

**아키텍처 변형들:**

| 변형 | 설명 | 특징 |
|------|------|------|
| 단일 헤드 (표준) | EOS 위치 은닉 상태 → 스칼라 | 가장 단순, 널리 사용 |
| 평균 풀링 헤드 | 모든 토큰 은닉 상태 평균 → 스칼라 | 길이에 덜 민감 |
| 토큰 수준 보상 | 각 토큰에 보상 부여 → 합산 | 세밀한 신호, 계산 비용 높음 |
| 앙상블 | 여러 독립 보상 모델의 통계 | 과최적화 완화 효과 |

**크기 선택**: 보상 모델은 일반적으로 정책 모델보다 같거나 작은 크기를 사용합니다. InstructGPT에서는 6B 보상 모델로 175B 정책을 학습했습니다. 보상 모델이 클수록 더 정확하지만, 추론 비용이 PPO 롤아웃마다 발생합니다.

## 보상 모델 품질의 중요성

보상 모델의 품질은 RLHF 전체 성능의 상한(upper bound)을 결정합니다. 불완전한 보상 모델은 다음과 같은 방식으로 최종 성능을 제한합니다:

- **편향(Bias)**: 체계적으로 잘못된 선호를 학습하면, PPO가 그 편향 방향으로 최적화됩니다
- **분산(Variance)**: 노이즈가 많은 보상 신호는 PPO 학습을 불안정하게 만듭니다
- **분포 외 일반화 실패**: 학습 분포를 벗어난 응답에 대해 부정확한 점수를 부여합니다

## Over-optimization 현상

**보상 모델 over-optimization** 은 PPO가 보상 모델 점수를 과도하게 최적화하여 실제 인간 선호와 괴리되는 현상입니다. Gao et al. (2022)은 이를 KL 발산 함수로 정량화했습니다.

Gao et al. (2022)의 **스케일링 법칙** 연구는 다음을 발견했습니다:
- 보상 모델 점수(proxy reward)는 KL 발산에 대해 $\sqrt{D_{KL}}$ 비율로 증가합니다
- 그러나 실제 인간 선호(gold reward)는 일정 KL을 넘으면 **역전**됩니다
- 이 역전 지점은 일반적으로 KL $\approx$ 10~20 nats 부근에서 발생합니다

$$r_{\text{gold}}(D_{KL}) \approx r_{\text{gold,0}} + a\sqrt{D_{KL}} - b \cdot D_{KL}$$

여기서 $a, b > 0$는 데이터·모델 의존 상수입니다. $D_{KL}$이 충분히 커지면 이차항이 지배하여 실제 보상이 감소합니다.

또한 보상 모델 크기가 클수록 over-optimization 내성이 높아집니다. 6B 보상 모델이 1B보다 더 높은 KL 수준에서도 gold reward와 일치합니다.

**완화 전략:**

| 전략 | 방법 | 장점 | 단점 |
|------|------|------|------|
| KL 페널티 | $\beta D_{KL}(\pi_\theta \| \pi_{ref})$ 추가 | 단순, 주류 방법 | $\beta$ 튜닝 필요 |
| 앙상블 (비관적 집계) | 여러 RM의 최솟값 사용 | over-opt 감소 | 계산 비용 증가 |
| 조기 중단 | 검증 지표 모니터링 | 간단 | 최적 시점 판단 어려움 |
| RM 주기적 재학습 | 새 정책 분포로 RM 업데이트 | 분포 이동 적응 | 비용 대폭 증가 |
| 보상 모델 앙상블 | Coste et al. 방법 | 이론적 근거 있음 | 학습 복잡도 증가 |

1. **KL 페널티**: $\beta D_{KL}(\pi_\theta \| \pi_{ref})$로 분포 이동 제한 (주류 방법)
2. **보상 모델 앙상블**: Coste et al. (2023)은 여러 보상 모델의 최솟값(pessimistic aggregation)을 사용하면 over-optimization이 감소함을 보였습니다
3. **조기 중단(Early Stopping)**: 보상 모델 점수와 다른 지표를 모니터링하여 적절한 시점에 학습 종료
4. **보상 모델 업데이트**: 학습 중 보상 모델을 주기적으로 재학습하여 분포 이동에 적응 (계산 비용 증가)

보상 모델의 한계는 RLHF 연구의 가장 활발한 주제 중 하나이며, RLAIF와 Constitutional AI는 이 한계를 다른 방식으로 접근하는 시도입니다.
