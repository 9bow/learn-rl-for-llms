## InstructGPT: RLHF의 이정표

Ouyang et al. (2022)의 **InstructGPT** 는 RLHF를 대규모 언어 모델에 성공적으로 적용하여, 이 분야의 이정표가 된 연구입니다. 이 논문은 이후 ChatGPT, GPT-4, Claude, Llama 2 등 거의 모든 주요 언어 모델의 정렬 방법론에 직접적인 영향을 미쳤습니다.

## 데이터 수집 방법론

InstructGPT는 OpenAI API를 통해 실제 사용자가 제출한 프롬프트를 기반으로 데이터를 구성했습니다. **40명의 계약직 주석자** 를 채용하여 세 가지 유형의 데이터를 수집했습니다:

**SFT 데이터 (~13,000개)**:
- 주석자가 프롬프트를 직접 작성하거나 API에서 샘플링
- 주석자가 고품질 응답을 직접 작성
- 세 가지 프롬프트 유형: 일반(plain), 소수샷(few-shot), 사용자 기반(user-based)

**RM 데이터 (~33,000개 비교 쌍)**:
- 동일 프롬프트에 대해 4~9개 모델 응답 생성
- 주석자가 모든 쌍의 선호도 순위 매김
- $\binom{K}{2}$개의 비교 쌍을 효율적으로 추출

**PPO 프롬프트 (~31,000개)**:
- API 사용자 프롬프트에서 샘플링
- 응답은 모델이 직접 생성 (주석 불필요)

## 3단계 학습의 상세 과정

**1단계 SFT**: GPT-3(175B)를 SFT 데이터로 미세조정. 16 에포크 학습, 단순 cross-entropy 손실. 결과 모델을 $\text{GPT-3}_{SFT}$라 칭함.

**2단계 RM**: $\text{GPT-3}_{SFT}$의 6B 버전을 보상 모델로 학습 (175B 대신 6B 사용 — 4개 모델 동시 유지 시 메모리 절감). Bradley-Terry 손실로 학습. 여러 순위를 한 번에 학습하는 효율적 배치 구성 사용.

**3단계 PPO**: $\text{GPT-3}_{SFT}$ (1.3B, 6B, 175B 세 크기)를 Actor로, 6B RM을 보상 모델로, SFT 모델을 레퍼런스로 PPO 학습. KL 계수 $\beta$는 적응형으로 조정.

## 핵심 실험 결과

InstructGPT의 가장 놀라운 발견은 **크기보다 정렬이 중요할 수 있다** 는 것입니다:

> 1.3B InstructGPT 모델이 175B GPT-3보다 인간 평가자에게 더 선호됨 (85% vs 71%, 각자 대비 기본값과 비교)

상세 결과:

| 비교 | InstructGPT 선호 비율 |
|------|---------------------|
| InstructGPT 1.3B vs GPT-3 175B | 85% |
| InstructGPT 175B vs GPT-3 175B | 97% |
| InstructGPT 175B vs GPT-3 175B SFT | 81% |

진실성(truthfulness) 측면에서도 InstructGPT는 TruthfulQA에서 GPT-3 대비 개선을 보였으나, 일부 NLP 벤치마크(SQuAD, HellaSwag 등)에서 소폭 하락 — **alignment tax** 의 실증적 관찰.

## ChatGPT와 GPT-4로의 진화

**ChatGPT(2022년 11월)**: InstructGPT의 후속으로, GPT-3.5 계열 모델에 RLHF를 적용. 대화 맥락 유지와 지시어 따르기에 특화. 구체적 학습 방법론은 공개되지 않았으나, InstructGPT 파이프라인을 기반으로 한 것으로 알려짐. 출시 5일 만에 100만 사용자 달성.

**GPT-4(2023년 3월)**: OpenAI Technical Report에 따르면, RLHF 외에도 "rule-based reward models(RBRMs)"를 추가 도입. 멀티모달(이미지+텍스트) 입력 지원. 더 정교한 안전 평가 체계(red-teaming, 외부 안전 연구자 참여). MMLU 86.4%, Bar Exam 90th percentile 달성.

## Claude의 Constitutional AI

Anthropic의 **Claude** 시리즈는 Constitutional AI(CAI) 방법론을 핵심으로 사용합니다. RLHF의 인간 유해성 레이블 대신 AI 피드백을 활용하여:

- Claude 1(2023): Constitutional AI 첫 공개 적용
- Claude 2(2023): 더 긴 컨텍스트(100K 토큰), 향상된 코딩 능력
- Claude 3(2024): Haiku/Sonnet/Opus 계층 구조, 멀티모달 지원

Claude의 헌법(constitution)은 UN 인권 선언, Anthropic 이용 약관, 그리고 다양한 AI 안전 원칙에서 도출된 수십 개의 자연어 원칙으로 구성됩니다.

## Llama 2의 RLHF

Meta의 **Llama 2**(Touvron et al., 2023)는 오픈소스 모델 중 RLHF 적용을 가장 상세히 공개한 사례입니다:

- **Ghost Attention(GAtt)**: 다중 턴 대화에서 시스템 프롬프트를 일관되게 따르는 새로운 기법
- **이중 보상 모델**: 유용성(helpfulness) RM과 안전성(safety) RM을 별도 학습
- **RLHF-V 반복**: 5번의 반복적 RLHF 적용으로 지속적 개선
- **100만 개 이상의 인간 선호 주석** 수집 (규모에서 InstructGPT 대비 30배 이상)

## 주요 RLHF 적용 모델 비교

| 모델 | 조직 | RLHF 방법 | 데이터 규모 | 공개 여부 |
|------|------|----------|------------|---------|
| InstructGPT | OpenAI | PPO-RLHF | ~77K 쌍 | 논문 공개 |
| ChatGPT | OpenAI | PPO-RLHF (추정) | 미공개 | 모델 공개 |
| GPT-4 | OpenAI | PPO-RLHF + RBRM | 미공개 | 기술 보고서 |
| Claude 1/2/3 | Anthropic | Constitutional AI | 미공개 | 논문 공개 |
| Llama 2 Chat | Meta | PPO-RLHF (이중 RM) | ~100만 쌍 | 모델+논문 |
| Gemini | Google | RLHF (추정) | 미공개 | 기술 보고서 |

InstructGPT는 RLHF라는 아이디어를 대규모 LLM에 처음으로 성공적으로 적용한 연구로서, 현재 AI 정렬 연구의 출발점이 되었습니다. 이후 등장한 모든 주요 언어 모델은 정도의 차이는 있지만 RLHF 또는 그 변형을 핵심 정렬 방법론으로 채택하고 있습니다.
