## 웹 에이전트의 도전

웹 내비게이션은 LLM 에이전트의 가장 실용적인 응용 분야 중 하나다. 사용자 대신 웹사이트를 탐색하고, 정보를 검색하고, 폼을 작성하고, 예약을 완료하는 작업이 포함됩니다. 그러나 이 태스크는 다음과 같은 이유로 매우 어렵습니다.

- **동적 환경**: 웹사이트 구조가 예측 불가능하게 변합니다.
- **긴 에피소드**: 하나의 태스크가 수십 번의 클릭과 입력을 요구합니다.
- **희소 보상**: 태스크 완료 여부만 알 수 있습니다.
- **부분 관측**: 현재 화면만 볼 수 있어 전체 구조를 알기 어렵습니다.

**WebArena** 벤치마크는 이런 실제적인 웹 태스크를 평가합니다. 초기 SOTA 모델들조차 10% 미만의 성공률을 보였다.

## WebRL: 자기 진화 온라인 커리큘럼 RL

**WebRL** (Qi et al., arXiv:2411.02337, ICLR 2025)은 웹 에이전트 RL의 핵심 문제인 **데이터 품질과 커리큘럼 설계** 를 해결합니다.

### 핵심 아이디어: 자기 진화 커리큘럼

WebRL의 핵심은 에이전트가 학습하면서 **스스로 난이도를 조절하는 커리큘럼** 을 만든다는 것입니다. 너무 쉬운 태스크는 학습 신호가 없고, 너무 어려운 태스크는 성공 경험이 없어 보상이 없습니다. 최적의 난이도는 에이전트가 성공할 수도 실패할 수도 있는 경계 근방입니다.

$$\text{난이도 목표}: P(\text{success}) \approx 0.5$$

실패한 태스크는 커리큘럼에서 더 쉬운 서브태스크로 분해됩니다. 성공한 태스크는 더 어려운 변형으로 교체됩니다.

### ORM 기반 보상 설계

WebRL은 **결과 보상 모델(ORM)** 을 학습하여 중간 단계에도 보상을 제공합니다. 이는 희소 보상 문제를 완화합니다.

$$r_t = \begin{cases} r_{ORM}(s_t, a_t) & \text{중간 스텝} \\ r_{final} & \text{에피소드 종료} \end{cases}$$

### 성능 결과

| 모델 | WebArena 성공률 |
|------|--------------|
| Llama-3.1-8B (SFT) | 4.8% |
| GPT-4-Turbo | 17.6% |
| Claude-3.5-Sonnet | 22.1% |
| **WebRL (Llama-3.1-8B)** | **42.4%** |

8B 모델이 GPT-4-Turbo를 2.4배 초과하는 놀라운 결과입니다. 자기 진화 커리큘럼의 효과가 모델 크기를 압도했다.

### 학습 안정화 기법

긴 에피소드에서의 RL 학습은 불안정하다. WebRL은 세 가지 기법으로 이를 안정화합니다.

1. **KL 발산 제약**: 정책이 급격히 변화하지 않도록 제한
2. **가치 함수 클리핑**: 가치 추정의 급격한 변화 방지
3. **배치 정규화**: 보상 스케일 정규화

$$\mathcal{L}_{WebRL} = \mathcal{L}_{PPO} + \beta_1 D_{KL} + \beta_2 \mathcal{L}_{value}$$

## WebAgent-R1: 엔드투엔드 멀티턴 RL

**WebAgent-R1** (arXiv:2505.16421)은 웹 에이전트 RL의 다른 측면에 집중합니다. **엔드투엔드 멀티턴 RL** 과 **비동기 생성** 입니다.

### 엔드투엔드 멀티턴 설계

기존 웹 에이전트 RL은 각 스텝을 독립적으로 최적화합니다. WebAgent-R1은 전체 에피소드를 하나의 멀티턴 대화로 처리하여 **엔드투엔드** 로 최적화합니다.

```
시스템 프롬프트: "당신은 웹 에이전트입니다..."
사용자: "Amazon에서 빨간 운동화를 찾아주세요"
에이전트: [클릭: 검색창] [입력: "빨간 운동화"] [클릭: 검색]
환경: [검색 결과 페이지 스크린샷]
에이전트: [클릭: 첫 번째 결과] ...
...
환경: [태스크 완료 확인]
보상: r = 1.0
```

이 전체 시퀀스를 하나의 RL 에피소드로 처리합니다.

### 비동기 생성

웹 에이전트 RL에서 병목은 두 가지다. 모델의 추론 시간과 웹 환경의 실행 시간입니다. 이 둘은 직렬로 실행하면 서로를 기다려야 합니다.

WebAgent-R1의 **비동기 생성** 은 이 둘을 병렬화합니다.

```
시간 →
모델: [추론 중] [추론 완료] [다음 추론 중] ...
환경:            [이전 행동 실행 중] [결과 반환] ...
```

이를 통해 GPU 활용률을 크게 높이고 훈련 속도를 향상시킨다.

### 이진 결과 보상

WebAgent-R1은 단순한 이진 보상(성공/실패)만 사용합니다. 복잡한 보상 엔지니어링 없이도 강력한 성능을 달성합니다.

$$r = \mathbf{1}[\text{태스크 완료}]$$

| 모델 | 파라미터 | WebArena 성공률 |
|------|---------|--------------|
| Qwen-2.5-3B (SFT) | 3B | 6.1% |
| GPT-4V | 폐쇄형 | 14.3% |
| **WebAgent-R1 (Qwen-2.5-3B)** | **3B** | **33.9%** |

3B 파라미터 모델이 GPT-4V를 2.4배 초과합니다.

## DigiRL: 안드로이드 디바이스 제어

**DigiRL** (Bai et al., NeurIPS 2024, arXiv:2406.11896)은 웹 내비게이션을 넘어 **안드로이드 스마트폰 제어** 라는 더 복잡한 에이전트 태스크를 다룬다.

### 2단계 RL: 오프라인 + 온라인

DigiRL은 두 단계의 RL을 순차적으로 적용합니다.

**1단계: 오프라인 RL (사전 학습)**

수집된 경험 데이터 $\mathcal{D}_{offline}$ 으로 정책을 초기화합니다. 실제 디바이스 없이도 학습 가능합니다.

$$\pi_1 = \text{OfflineRL}(\pi_0, \mathcal{D}_{offline})$$

**2단계: 온라인 RL (미세조정)**

실제 안드로이드 에뮬레이터와 상호작용하며 추가 학습합니다.

$$\pi^* = \text{OnlineRL}(\pi_1, \text{Emulator})$$

이 2단계 접근법은 오프라인 RL의 안정성과 온라인 RL의 환경 적응성을 결합합니다.

### VLM 기반 에이전트

DigiRL은 **1.5B 파라미터 VLM** 을 에이전트로 사용합니다. 스크린샷을 보고 어떤 행동을 취할지 결정합니다.

행동 공간은 다음과 같이 구성됩니다.
- `tap(x, y)`: 화면 좌표 탭
- `swipe(x1, y1, x2, y2)`: 스와이프
- `type(text)`: 텍스트 입력
- `back()`: 뒤로 가기
- `home()`: 홈 화면

### 성능 비교

| 모델 | 파라미터 | 안드로이드 성공률 |
|------|---------|--------------|
| GPT-4V | 폐쇄형 | 8.3% |
| LLaMA-2-70B | 70B | 12.1% |
| DigiRL-1단계만 | 1.5B | 41.2% |
| **DigiRL (2단계)** | **1.5B** | **67.2%** |

1.5B VLM이 GPT-4V를 8배 초과하는 성능을 달성했다. 이는 RL 훈련의 효과가 모델 크기를 압도적으로 능가함을 보여줍니다.

## 웹 에이전트 RL 성능 비교

| 방법 | 대상 환경 | 모델 크기 | 기준선 대비 향상 | 핵심 혁신 |
|------|---------|---------|--------------|---------|
| WebRL | WebArena | 8B | 8.8배 | 자기 진화 커리큘럼 |
| WebAgent-R1 | WebArena | 3B | 5.6배 | 비동기 멀티턴 RL |
| DigiRL | 안드로이드 | 1.5B | 8.1배 | 오프라인+온라인 2단계 |

세 방법 모두 기준선 대비 5~9배의 성능 향상을 달성했다. RL이 웹 에이전트 분야에서 단순히 유용한 것이 아니라 **게임 체인저** 수준의 혁신임을 보여줍니다.

## 주요 설계 원칙

웹 에이전트 RL에서 공통적으로 중요한 설계 원칙들이 있습니다.

**환경 현실성**: 실제 웹사이트 구조와 유사한 환경에서 훈련해야 실제 성능이 좋습니다.

**커리큘럼 설계**: 에이전트의 현재 능력에 맞는 난이도의 태스크를 제공해야 효율적인 학습이 가능합니다.

**보상 신호 풍부화**: 최종 성공만 보상하면 학습이 느립니다. 중간 단계 보상이나 부분 완료 보상이 도움이 됩니다.

**계산 효율**: 비동기 생성, 병렬 환경 실행 등으로 훈련 비용을 줄여야 실용적입니다.

## 정리

WebRL, WebAgent-R1, DigiRL은 웹 내비게이션과 디바이스 제어에서 RL의 파괴적 효과를 입증했다. 8B 이하의 소형 모델이 GPT-4V와 같은 최첨단 폐쇄형 모델을 수 배 초과하는 성능을 달성했다. 자기 진화 커리큘럼, 비동기 멀티턴 RL, 2단계 오프라인-온라인 RL이라는 각자의 핵심 혁신은 모두 에이전트 RL의 근본 도전인 데이터 효율성, 훈련 안정성, 환경 적응성을 다른 각도에서 해결합니다.
