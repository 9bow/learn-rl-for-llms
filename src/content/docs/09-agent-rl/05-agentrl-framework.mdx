## 에이전트 RL 프레임워크의 필요성

개별 에이전트 RL 연구(WebRL, DigiRL 등)들은 각자의 문제를 해결했지만, 일반적인 에이전트 RL 프레임워크는 부족했다. 여러 도메인, 여러 태스크, 여러 RL 알고리즘을 통합하는 범용 시스템이 필요합니다. 이 섹션은 그런 통합 프레임워크들을 다룬다.

## AgentRL: 비동기 생성-학습 파이프라인

**AgentRL** (arXiv:2510.04206, NeurIPS 2024 spotlight)은 오픈소스 LLM으로 GPT-4o, Claude-3.5-Sonnet, DeepSeek-R1을 5개 에이전트 태스크에서 초과하는 성능을 달성한 프레임워큽니다.

### 비동기 생성-학습 파이프라인

기존 RL 훈련은 **동기적(synchronous)** 입니다. 샘플 생성 → 환경 실행 → 업데이트를 순서대로 수행합니다. 이 방식은 병목이 많습니다.

```
[동기적]
생성 → 환경 → 업데이트 → 생성 → 환경 → 업데이트 → ...
GPU 낭비: 환경 실행 중 GPU 유휴

[비동기적 - AgentRL]
생성기 1 → 환경 1 ↘
생성기 2 → 환경 2  → 버퍼 → 업데이트 → 생성기 갱신
생성기 3 → 환경 3 ↗
GPU 활용: 항상 학습 중
```

비동기 파이프라인은 GPU 활용률을 크게 높이고 훈련 속도를 향상시킨다.

### Cross-Policy 샘플링

AgentRL의 핵심 혁신 중 하나는 **cross-policy 샘플링** 입니다. 현재 정책 $\pi_\theta$ 만으로 샘플링하면 탐색 다양성이 제한됩니다. Cross-policy 샘플링은 여러 이전 정책 체크포인트에서도 샘플을 수집합니다.

$$\mathcal{D} = \{o \sim \pi_\theta\} \cup \{o \sim \pi_{\theta-1}\} \cup \{o \sim \pi_{\theta-2}\}$$

중요도 샘플링(importance sampling)으로 오래된 정책의 샘플도 현재 정책 학습에 활용합니다.

$$w_i = \frac{\pi_\theta(o_i)}{\pi_{old}(o_i)}$$

이를 통해 탐색 다양성을 높이면서도 안정적인 학습이 가능합니다.

### 과제 Advantage 정규화

서로 다른 난이도의 과제를 동시에 학습할 때, 쉬운 과제의 advantage가 어려운 과제의 advantage를 압도하는 문제가 발생합니다. AgentRL은 **과제별 advantage 정규화** 로 이를 해결합니다.

$$\hat{A}_t^{task} = \frac{A_t - \mu_{task}}{\sigma_{task} + \epsilon}$$

각 과제 내에서 별도로 정규화함으로써 모든 과제에서 균형 있는 학습이 가능합니다.

### 성능 결과

| 과제 | GPT-4o | Claude-3.5-Sonnet | DeepSeek-R1 | AgentRL |
|------|--------|------------------|------------|---------|
| WebArena | 38.2% | 32.7% | 41.3% | **54.6%** |
| SWE-Bench | 22.1% | 26.8% | 24.7% | **33.2%** |
| OSWorld | 18.4% | 21.3% | 19.8% | **28.7%** |
| Tool Use | 67.3% | 71.2% | 69.8% | **81.4%** |
| Coding | 72.1% | 68.4% | 75.3% | **84.2%** |

오픈소스 모델 기반 AgentRL이 모든 5개 과제에서 GPT-4o와 Claude-3.5-Sonnet을 초과했다.

## Agent-R1: 멀티턴 RL 훈련 플랫폼

**Agent-R1** (arXiv:2511.14460)은 특정 에이전트 태스크에 특화된 것이 아니라 **범용 멀티턴 RL 훈련 플랫폼** 을 목표로 합니다.

### 플랫폼 아키텍처

```
Agent-R1 플랫폼
├── 환경 인터페이스
│   ├── WebArena
│   ├── ALFWorld
│   ├── ScienceWorld
│   └── 커스텀 환경
├── RL 알고리즘 모듈
│   ├── PPO
│   ├── GRPO
│   ├── DPO (오프라인)
│   └── REINFORCE
├── 보상 모듈
│   ├── 환경 보상
│   ├── 형식 보상
│   └── 커스텀 보상
└── 모델 인터페이스
    ├── Qwen 시리즈
    ├── LLaMA 시리즈
    └── 기타 HuggingFace 모델
```

### 멀티턴 RL 지원

Agent-R1은 멀티턴 대화를 자연스럽게 지원합니다. 각 턴을 독립적인 RL 스텝으로 처리하되, 이전 턴의 정보가 상태에 포함됩니다.

**상태 표현:**

$$s_t = [\text{시스템 프롬프트}, u_1, a_1, \ldots, u_t]$$

**보상 집계:**

$$R_{total} = \sum_{t=1}^{T} \gamma^{T-t} r_t$$

나중 턴의 보상에 더 높은 가중치를 부여하여 대화 완결을 장려합니다.

### 다양한 RL 알고리즘 비교

Agent-R1 플랫폼에서 다양한 RL 알고리즘을 동일한 조건으로 비교한 결과:

| 알고리즘 | WebArena | ScienceWorld | 훈련 안정성 | 샘플 효율 |
|---------|---------|------------|---------|---------|
| PPO | 41.2% | 67.3% | 높음 | 중간 |
| GRPO | 43.7% | 71.2% | 중간 | 높음 |
| REINFORCE | 38.4% | 62.1% | 낮음 | 낮음 |
| DPO (오프라인) | 35.2% | 58.7% | 높음 | 낮음 |

GRPO가 대부분의 에이전트 태스크에서 최고 성능을 보이면서도 합리적인 안정성을 제공했다.

## Agent²: 에이전트가 에이전트를 생성

**Agent²** (arXiv:2509.13368)는 메타 에이전트(meta-agent) 패러다임을 도입합니다. 에이전트가 다른 에이전트를 생성하고 조율하는 계층적 구조입니다.

### 메타 에이전트 패러다임

```
메타 에이전트 (Agent²)
    │ 태스크 분석 및 에이전트 설계
    ├── 하위 에이전트 A: "웹 검색 전문"
    ├── 하위 에이전트 B: "코드 작성 전문"
    ├── 하위 에이전트 C: "데이터 분석 전문"
    └── 하위 에이전트 D: "결과 검증 전문"
    │ 결과 통합 및 최종 출력
```

**메타 에이전트의 역할:**
1. 복잡한 태스크를 서브태스크로 분해
2. 각 서브태스크에 적합한 하위 에이전트 생성
3. 하위 에이전트들의 실행 조율
4. 결과 통합 및 품질 검증

### 에이전트 생성의 RL 최적화

메타 에이전트가 어떤 하위 에이전트를 어떻게 생성할지를 RL로 최적화합니다.

$$\pi_{meta}^* = \arg\max_{\pi_{meta}} \mathbb{E}\left[r_{final} | \{a_1, \ldots, a_K\} \sim \pi_{meta}\right]$$

여기서 $\{a_1, \ldots, a_K\}$ 는 생성된 $K$ 개의 하위 에이전트 명세다.

### 생성 가능한 에이전트 유형

| 에이전트 유형 | 특화 영역 | 생성 조건 |
|-----------|---------|---------|
| 연구 에이전트 | 정보 수집, 검색 | 불확실한 정보 필요 시 |
| 실행 에이전트 | 코드 작성, 도구 사용 | 구체적 행동 필요 시 |
| 검증 에이전트 | 결과 확인, 테스트 | 품질 검증 필요 시 |
| 전문가 에이전트 | 도메인 특화 지식 | 전문 지식 필요 시 |

### Agent² vs 단일 에이전트

| 특성 | 단일 에이전트 | Agent² |
|------|----------|-------|
| 태스크 복잡성 처리 | 제한적 | 높음 |
| 병렬 실행 | 불가 | 가능 |
| 전문화 | 일반적 | 전문화 가능 |
| 조율 오버헤드 | 없음 | 있음 |
| 오류 격리 | 어려움 | 쉬움 |
| 계산 비용 | 낮음 | 높음 |

## 에이전트 RL 프레임워크 비교

| 특성 | AgentRL | Agent-R1 | Agent² |
|------|--------|---------|-------|
| 핵심 혁신 | 비동기 파이프라인 | 범용 플랫폼 | 메타 에이전트 |
| 대상 | 단일 에이전트 | 단일/멀티턴 | 다중 에이전트 |
| RL 알고리즘 | PPO 기반 | PPO/GRPO/DPO | 커스텀 |
| 확장성 | 높음 | 높음 | 매우 높음 |
| 구현 복잡도 | 중간 | 낮음 | 높음 |
| 공개 수준 | 논문 + 코드 | 완전 공개 | 논문 공개 |

## 정리

AgentRL, Agent-R1, Agent²는 에이전트 RL을 개별 연구에서 범용 프레임워크로 발전시킨 중요한 기여입니다. AgentRL의 비동기 생성-학습 파이프라인과 cross-policy 샘플링은 오픈 LLM으로 GPT-4o와 Claude-3.5-Sonnet을 초과하는 성능을 가능하게 했다. Agent-R1은 다양한 RL 알고리즘과 환경을 통합하는 범용 플랫폼을 제공하며, Agent²는 에이전트가 에이전트를 생성하는 메타 에이전트 패러다임으로 복잡한 태스크 처리 능력을 크게 확장했다.
