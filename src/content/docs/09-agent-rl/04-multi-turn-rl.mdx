## 멀티턴 RL의 필요성

단일 턴(single-turn) RL은 하나의 질문에 하나의 답변을 최적화합니다. 그러나 실제 LLM 응용의 대부분은 **멀티턴 대화** 다. 고객 서비스, 코드 디버깅, 교육, 협상 등은 모두 여러 번의 상호작용을 통해 목표를 달성합니다.

멀티턴 RL은 이러한 상호작용 전체를 최적화합니다. 개별 응답의 품질이 아니라 **대화 전체의 결과** 를 최적화 목표로 삼는다.

### 멀티턴 MDP 공식화

멀티턴 대화를 MDP로 모델링하면 다음과 같습니다.

- **상태** $s_t$: 턴 $t$ 까지의 대화 이력 $\{(u_1, a_1), \ldots, (u_t, a_t)\}$
- **행동** $a_t$: 현재 턴의 LLM 응답
- **전환**: $s_{t+1} = s_t \cup \{(u_{t+1}, a_t)\}$ (사용자의 다음 메시지 포함)
- **보상** $r_T$: 대화 완료 시 최종 보상

$$V^\pi(s_t) = \mathbb{E}_\pi\left[\sum_{k=t}^{T} \gamma^{k-t} r_k \middle| s_t\right]$$

## 멀티턴 RL의 핵심 도전

### 1. 신용 할당(Credit Assignment) 문제

$T$ 턴의 대화에서 최종 보상이 $r_T$ 로만 주어진다면, 어떤 턴의 응답이 성공/실패에 기여했는지 판단하기 어렵습니다. 3번째 턴의 잘못된 설명이 7번째 턴의 사용자 이탈을 유발했을 수도 있습니다.

$$\delta_t = r_T \cdot \mathbf{1}[t = T] + \gamma V(s_{t+1}) - V(s_t)$$

이 시간차 오류는 초기 턴에서 거의 0에 가까워지는 문제가 있습니다.

**해결 접근법:**

- **밀집 보상(Dense Reward)**: 각 턴마다 중간 보상 제공
- **역방향 크레딧**: 최종 보상을 역방향으로 각 턴에 분배
- **턴 수준 가치 함수**: 각 턴의 장기적 가치를 별도로 추정

### 2. 긴 컨텍스트 처리

멀티턴 대화는 컨텍스트가 길다. 10턴 대화에서 각 턴이 200 토큰이면 2,000 토큰의 컨텍스트가 생성됩니다. 이는 메모리와 계산 비용을 크게 증가시킨다.

$$\text{비용} \propto T_{context}^2$$

어텐션 메커니즘의 이차적 복잡도가 긴 컨텍스트에서 훈련을 비효율적으로 만듭니다.

### 3. 사용자 모델링

멀티턴 RL에서 사용자의 다음 메시지는 에이전트의 행동에 의존합니다. 그러나 실제 사용자 행동은 복잡하고 예측하기 어렵습니다. 훈련 시에는 **사용자 시뮬레이터** 가 필요하며, 이 시뮬레이터의 품질이 학습된 정책의 품질을 결정합니다.

## LMRL Gym: 멀티턴 RL 벤치마크

**LMRL Gym** (Abdulhai et al., NeurIPS 2024 / ICML 2025)은 LLM의 멀티턴 RL 능력을 평가하기 위한 포괄적 벤치마큽니다. 8개의 다양한 과제로 구성됩니다.

### 대화 과제 (3개)

| 과제 | 설명 | 성공 기준 |
|------|------|---------|
| Twenty Questions | 스무고개로 개념 맞추기 | 20턴 내 정답 |
| Guess My City | 질문으로 도시 추측 | 10턴 내 정답 |
| Persuasion | 가상 기부 설득 | 목표 금액 달성 |

### 게임 과제 (5개)

| 과제 | 설명 | 에피소드 길이 |
|------|------|------------|
| Wordle | 단어 추측 게임 | 최대 6턴 |
| Car Dealer | 가격 협상 | 가변 |
| Prisoner's Dilemma | 반복 죄수의 딜레마 | 10라운드 |
| Negotiation | 다자간 협상 | 가변 |
| Diplomacy | 전략 보드 게임 | 다중 에이전트 |

### 기준선 비교

| 모델 | 대화 과제 평균 | 게임 과제 평균 |
|------|------------|------------|
| GPT-3.5 (zero-shot) | 34.2% | 41.7% |
| GPT-4 (zero-shot) | 52.8% | 58.3% |
| LLaMA-2-7B (SFT) | 28.1% | 33.4% |
| LLaMA-2-7B (멀티턴 RL) | **47.3%** | **54.6%** |

멀티턴 RL로 학습된 7B 모델이 GPT-3.5를 초과했다.

## Context-lite 멀티턴 RL

**Context-lite 멀티턴 RL** 은 긴 컨텍스트 처리 비용 문제를 해결합니다. 전체 대화 이력 대신 **압축된 상태 표현** 을 사용합니다.

### 상태 압축

긴 대화 이력을 간결한 상태 벡터로 압축합니다.

$$\tilde{s}_t = \text{Compress}(s_t) = \text{Summarizer}(\{(u_1, a_1), \ldots, (u_t, a_t)\})$$

이 요약은 현재 목표 달성에 관련된 핵심 정보만 포함합니다.

**장점:**
- 컨텍스트 길이를 $O(T)$ 에서 $O(1)$ 로 감소
- 메모리 효율 대폭 향상
- 더 긴 대화에 적용 가능

**단점:**
- 압축 과정에서 정보 손실
- 요약 모델의 품질에 의존

## 멀티턴 RLHF를 위한 효율적 정책 최적화

**Efficient Multi-Turn RLHF** (Xiong et al., arXiv:2410.04612)는 멀티턴 대화에서 RLHF를 효율적으로 적용하는 방법을 제안합니다.

### 턴 수준 보상 분해

전체 대화 보상을 턴 수준으로 분해합니다.

$$r_T = \sum_{t=1}^{T} \alpha_t \cdot r_t^{turn}$$

여기서 $\alpha_t$ 는 각 턴의 중요도 가중치이고, $r_t^{turn}$ 은 턴 $t$ 의 응답 품질입니다.

### 선호 데이터 수집 전략

멀티턴 선호 데이터 수집은 단일 턴보다 복잡하다. **턴 수준 분기(branch)** 전략을 사용합니다.

```
대화 이력: u₁ → a₁ → u₂ →
                          ├── a₂⁺ (선호 응답)
                          └── a₂⁻ (비선호 응답)
                          → 이후 대화 계속
```

동일한 대화 맥락에서 두 가지 응답 후보를 생성하고, 이후 대화 결과로 어떤 응답이 더 나은지 판단합니다.

### 손실 함수

$$\mathcal{L}_{MT-RLHF} = -\sum_{t=1}^{T} \mathbb{E}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(a_t^+ | s_t)}{\pi_{ref}(a_t^+ | s_t)} - \beta \log \frac{\pi_\theta(a_t^- | s_t)}{\pi_{ref}(a_t^- | s_t)}\right)\right]$$

이는 각 턴에서 선호 응답의 확률을 높이는 방향으로 학습합니다.

## 페르소나 일관성을 위한 멀티턴 RL

**페르소나 일관성 멀티턴 RL** (Bai et al., ICLR 2025)은 LLM이 긴 대화에서 일관된 성격과 가치관을 유지하도록 RL을 적용합니다.

### 페르소나 불일관성 문제

LLM은 긴 대화에서 자신의 이전 응답과 모순되는 답변을 하는 경향이 있습니다.

```
턴 3: "저는 채식주의자입니다"
턴 8: "스테이크 맛있게 먹었어요"  ← 불일관성!
```

이런 불일관성은 사용자 신뢰를 저하시키고 AI 어시스턴트의 유용성을 감소시킨다.

### 일관성 보상 설계

페르소나 일관성을 보상으로 정의합니다.

$$r_{consistency} = 1 - \frac{\text{count}(\text{contradictions})}{\text{count}(\text{total\_claims})}$$

모순 탐지는 별도의 NLI(Natural Language Inference) 모델로 수행합니다.

$$\text{contradiction} = \mathbf{1}[\text{NLI}(a_t, a_{t'}) = \text{CONTRADICTION}], \quad t \neq t'$$

### 성능 결과

멀티턴 RL 적용 후 페르소나 불일관성이 크게 감소했다.

| 모델 | 불일관성 비율 | 대화 품질 |
|------|-----------|---------|
| LLaMA-2-7B (기준) | 18.3% | 3.2/5.0 |
| LLaMA-2-7B (SFT) | 12.7% | 3.6/5.0 |
| **LLaMA-2-7B (멀티턴 RL)** | **8.1%** | **4.1/5.0** |

멀티턴 RL로 불일관성을 55% 이상 감소시키고 대화 품질도 향상시켰다.

## 멀티턴 RL 방법 비교

| 방법 | 핵심 문제 | 접근법 | 주요 결과 |
|------|---------|-------|---------|
| LMRL Gym | 벤치마크 부재 | 8개 과제 표준화 | 7B RL > GPT-3.5 |
| Context-lite | 긴 컨텍스트 비용 | 상태 압축 | O(T) → O(1) |
| Efficient MT-RLHF | 선호 데이터 수집 | 턴 수준 분기 | 효율적 RLHF |
| 페르소나 일관성 | 대화 모순 | 일관성 보상 | 불일관성 55% 감소 |

## 정리

멀티턴 RL은 단일 응답 최적화를 넘어 대화 전체를 목표로 학습하는 패러다임입니다. 신용 할당 문제, 긴 컨텍스트 처리, 사용자 모델링이 핵심 도전이며, LMRL Gym은 이를 표준화된 방식으로 평가합니다. Context-lite 접근법으로 컨텍스트 비용을 줄이고, 효율적 RLHF로 선호 학습을 최적화하며, 페르소나 일관성 RL로 대화 신뢰성을 높이는 방향으로 이 분야는 빠르게 발전하고 있습니다.
