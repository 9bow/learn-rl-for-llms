## 멀티 에이전트 시스템으로의 확장

단일 LLM 에이전트는 복잡한 태스크에서 한계가 있습니다. 인간 사회처럼 **여러 에이전트가 협력** 하면 더 어려운 문제를 해결할 수 있습니다. 연구자, 작가, 비평가, 코더가 각자의 역할로 협력하듯, LLM 에이전트들도 역할을 분담하고 상호작용할 수 있습니다.

그러나 멀티 에이전트 시스템에서 RL은 단일 에이전트보다 훨씬 복잡하다. 각 에이전트의 행동이 다른 에이전트의 보상에 영향을 미치는 **비정상성(non-stationarity)** 문제가 핵심 도전입니다.

### 멀티 에이전트 MDP(MAMDP)

$N$ 개의 에이전트로 구성된 시스템에서 각 에이전트 $i$ 는 자신의 정책 $\pi_i$ 를 가지며, 보상은 모든 에이전트의 행동에 의존합니다.

$$r_i(s_t, \mathbf{a}_t) = r_i(s_t, a_t^1, a_t^2, \ldots, a_t^N)$$

에이전트 $i$ 의 최적 정책은 다른 에이전트들의 정책에 의존합니다.

$$\pi_i^* = \arg\max_{\pi_i} \mathbb{E}\left[\sum_t \gamma^t r_i(s_t, a_t^i, a_t^{-i})\right]$$

여기서 $a_t^{-i}$ 는 에이전트 $i$ 를 제외한 모든 에이전트의 행동입니다. 다른 에이전트들도 동시에 학습하므로 환경이 계속 변합니다.

## MARFT: 멀티 에이전트 강화 미세조정

**MARFT** (arXiv:2504.16129)는 여러 LLM 에이전트를 **동시에 강화학습으로 미세조정** 하는 프레임워큽니다.

### 핵심 설계

MARFT는 세 가지 유형의 에이전트 관계를 지원합니다.

**완전 협력(Fully Cooperative)**: 모든 에이전트가 공동 목표를 위해 협력

$$r_1 = r_2 = \ldots = r_N = r_{team}$$

**완전 경쟁(Fully Competitive)**: 제로섬 게임 구조

$$\sum_{i=1}^{N} r_i = 0$$

**혼합(Mixed)**: 협력과 경쟁이 공존

$$r_i = r_i^{team} + r_i^{individual}$$

### 중앙화 훈련, 분산화 실행 (CTDE)

MARFT는 **CTDE(Centralized Training, Decentralized Execution)** 패러다임을 따릅니다. 훈련 시에는 모든 에이전트의 정보를 중앙에서 활용하지만, 실행 시에는 각 에이전트가 자신의 관찰만으로 결정합니다.

$$\pi_i(a_i | o_i) \leftarrow \text{훈련 시}: Q(s, a_1, \ldots, a_N)$$

중앙화된 가치 함수 $Q(s, \mathbf{a})$ 를 사용해 각 에이전트의 기여도를 정확히 추정합니다.

### MARFT의 보상 분해

공동 보상 $r_{team}$ 을 각 에이전트에게 공정하게 분배하는 것이 핵심입니다. MARFT는 **Shapley 값** 기반 분해를 사용합니다.

$$\phi_i = \sum_{S \subseteq N \setminus \{i\}} \frac{|S|!(N-|S|-1)!}{N!} [v(S \cup \{i\}) - v(S)]$$

여기서 $\phi_i$ 는 에이전트 $i$ 의 기여도, $v(S)$ 는 에이전트 부분집합 $S$ 의 가치다.

### 적용 사례: 소프트웨어 개발

| 역할 | 에이전트 | 책임 |
|------|---------|------|
| 기획자 | Agent 1 | 요구사항 분석, 설계 |
| 개발자 | Agent 2, 3 | 코드 구현 |
| 테스터 | Agent 4 | 테스트 작성, 실행 |
| 리뷰어 | Agent 5 | 코드 리뷰 |

MARFT로 훈련된 이 시스템은 단일 에이전트 대비 복잡한 소프트웨어 프로젝트에서 37.2% 높은 완성도를 달성했다.

## LLM 협업과 멀티 에이전트 RL 역학

**LLM 협업 RL** (arXiv:2508.04652)은 다수 LLM 에이전트 간의 RL 역학을 체계적으로 분석합니다.

### 협업 설정에서의 학습

협업 태스크에서 에이전트들은 서로의 전문성을 보완하는 방향으로 분화(specialization)됩니다.

```
초기: 모든 에이전트가 동일한 일반 전략 사용
    ↓ (RL 학습)
수렴: 에이전트들이 서로 다른 역할에 특화
    - Agent A: 계획/전략에 집중
    - Agent B: 실행/구현에 집중
    - Agent C: 검증/피드백에 집중
```

이 **창발적 역할 분화(emergent role specialization)** 는 명시적으로 설계하지 않아도 RL을 통해 자연스럽게 발생합니다.

### 경쟁 설정에서의 역학

경쟁 설정에서는 에이전트들이 서로의 전략에 적응하며 공동 진화(co-evolution)합니다.

$$\pi_i^{t+1} = \text{BestResponse}(\pi_{-i}^t)$$

이 과정은 내시 균형(Nash Equilibrium)으로 수렴할 수도 있지만, 순환(cycling)이 발생하기도 합니다.

**순환 문제 해결책:**

- **Population 기반 훈련**: 다양한 과거 정책 풀을 유지하여 특정 전략에 과적합 방지
- **역할 랜덤화**: 매 에피소드마다 에이전트 역할을 무작위 배정
- **리그 훈련**: 다양한 수준의 에이전트들과 번갈아 대전

### 혼합 협력-경쟁 설정

현실적인 시나리오는 순수 협력이나 순수 경쟁이 아닌 혼합입니다. 팀 내 협력 + 팀 간 경쟁이 그 예다.

$$r_i = \underbrace{\alpha \cdot r_{team}}_{\text{팀 내 협력}} + \underbrace{(1-\alpha) \cdot r_{individual}}_{\text{팀 간 경쟁}}$$

$\alpha$ 값에 따라 협력과 경쟁의 균형이 결정됩니다.

## 사회적 딜레마와 LLM 에이전트

LLM 에이전트들을 **사회적 딜레마(social dilemma)** 에 놓으면 흥미로운 행동 패턴이 나타납니다.

### 죄수의 딜레마

두 에이전트가 협력(C) 또는 배반(D)을 선택할 때의 보수 행렬:

|  | 협력 (C) | 배반 (D) |
|--|---------|---------|
| **협력 (C)** | (3, 3) | (0, 5) |
| **배반 (D)** | (5, 0) | (1, 1) |

단기 최적은 배반이지만, 반복 게임에서는 **팃포탯(Tit-for-Tat)** 같은 협력 전략이 진화합니다.

RL로 훈련된 LLM 에이전트는 이 전략을 자연스럽게 학습합니다. 특히 상대방의 이전 행동 이력을 컨텍스트로 받을 때 더 정교한 조건부 협력 전략이 나타납니다.

### 공유지의 비극(Tragedy of the Commons)

공유 자원을 여러 에이전트가 활용하는 시나리오에서 각 에이전트의 단기 합리적 선택이 전체적 자원 고갈로 이어지는 문제입니다.

RL 훈련을 통해 에이전트들이 자원 보존의 장기적 이익을 학습할 수 있음이 확인되었다. 특히 에이전트 간 커뮤니케이션이 허용될 때 협력적 자원 관리가 창발합니다.

## 에이전트 간 커뮤니케이션 학습

멀티 에이전트 RL에서 에이전트들이 **커뮤니케이션 프로토콜** 을 스스로 학습하는 현상이 관찰됩니다.

### 창발적 언어

명시적으로 설계하지 않아도 에이전트들은 효율적인 정보 교환을 위한 언어를 개발합니다. 이 언어는 인간 언어와 다를 수 있지만 일관된 의미 체계를 갖습니다.

$$m_i^t = \text{Communicate}(o_i^t, h_i^t)$$

에이전트 $i$ 는 현재 관찰 $o_i^t$ 와 내부 상태 $h_i^t$ 를 바탕으로 메시지 $m_i^t$ 를 생성합니다. 이 메시지가 다른 에이전트들의 행동에 영향을 미친다.

### LLM 기반 커뮤니케이션의 특수성

LLM 에이전트는 자연어로 커뮤니케이션하므로 메시지가 해석 가능합니다. 이는 기존 강화학습의 창발적 언어와 달리 인간이 이해하고 검증할 수 있다는 장점이 있습니다.

| 커뮤니케이션 유형 | 예시 | 효과 |
|-------------|------|------|
| 상태 공유 | "현재 3번 태스크를 처리 중" | 중복 작업 방지 |
| 계획 공유 | "다음에 파일 A를 처리할 예정" | 의존성 관리 |
| 결과 공유 | "서브태스크 2 완료, 결과는..." | 정보 통합 |
| 도움 요청 | "이 부분 처리 방법 모름" | 전문성 활용 |

## 미래 전망: Emergent Behavior

멀티 에이전트 RL에서 가장 흥미로운 방향은 **창발적 행동(emergent behavior)** 입니다. 개별 에이전트 수준에서 설계하지 않은 집단 지성이 시스템 수준에서 나타나는 현상입니다.

### 관찰된 창발적 현상

| 현상 | 설명 | 조건 |
|------|------|------|
| 역할 분화 | 자발적 전문화 | 협력 태스크 |
| 언어 발달 | 효율적 커뮤니케이션 | 정보 교환 허용 |
| 리더십 창발 | 한 에이전트가 조율 | 복잡한 협력 |
| 사회 규범 | 암묵적 규칙 형성 | 반복 상호작용 |
| 집단 기억 | 공동 지식 저장소 | 공유 컨텍스트 |

### 스케일 효과

에이전트 수가 증가할수록 창발적 행동이 더 풍부해진다.

$$\text{창발 복잡도} \propto N^2$$

그러나 조율 비용도 함께 증가합니다.

$$\text{조율 비용} \propto N \log N$$

최적 에이전트 수는 태스크 복잡도와 이 두 요인의 균형에 따라 결정됩니다.

## 정리

멀티 에이전트 RL은 단일 에이전트 RL에서 해결하기 어려운 복잡한 태스크를 위한 자연스러운 확장입니다. MARFT는 여러 LLM 에이전트를 동시에 강화학습으로 훈련하는 체계적 프레임워크를 제공하고, LLM 협업 RL 연구는 협력과 경쟁 설정에서의 역학을 분석합니다. 사회적 딜레마에서의 협력 전략, 에이전트 간 커뮤니케이션 학습, 창발적 행동은 멀티 에이전트 LLM 시스템이 단순한 도구를 넘어 복잡한 사회적 역학을 가진 자율 시스템으로 발전할 가능성을 보여줍니다. 이 방향은 AI 안전성과 정렬(alignment) 연구에도 중요한 함의를 갖습니다.
