## LLM 에이전트란 무엇인가

**LLM 에이전트(LLM Agent)** 는 언어 모델을 핵심 추론 엔진으로 사용하여 환경과 상호작용하고 목표를 달성하는 자율 시스템입니다. 단순한 질문-답변 시스템과 달리, 에이전트는 **행동의 결과를 관찰하고 다음 행동을 조정** 합니다.

### 에이전트의 구성 요소

```
환경 (Environment)
    ↕ 관찰 / 행동
  에이전트
  ├── 인식 (Perception): 환경 상태를 텍스트/이미지로 변환
  ├── 기억 (Memory): 과거 관찰과 행동 기록
  ├── 계획 (Planning): 목표 달성을 위한 행동 계획
  ├── 행동 (Action): 도구 호출, 텍스트 생성, 코드 실행
  └── 반성 (Reflection): 결과 평가 및 전략 수정
```

에이전트가 상호작용하는 **환경** 은 매우 다양하다.

| 환경 유형 | 예시 | 행동 공간 |
|---------|------|---------|
| 웹 브라우저 | 검색, 클릭, 입력 | 이산 + 연속 |
| 코드 실행기 | 파이썬 실행, 파일 조작 | 텍스트 |
| 데이터베이스 | SQL 쿼리, 데이터 수정 | 텍스트 |
| 로봇 제어 | 모터 명령, 경로 계획 | 연속 |
| 소셜 미디어 | 게시, 검색, 팔로우 | 이산 |
| 게임 환경 | 이동, 공격, 상호작용 | 이산 |

## SFT 기반 에이전트의 한계

기존 LLM 에이전트는 **지도 학습(SFT)** 으로 훈련됩니다. 인간 전문가가 시연한 궤적(trajectory)을 모방 학습하는 방식입니다. 이 방식은 세 가지 구조적 한계를 갖습니다.

### 1. 시범 데이터 의존성

SFT는 고품질 시범 데이터가 반드시 필요합니다. 그러나 복잡한 에이전트 태스크에서 고품질 시범을 수집하는 것은 매우 비용이 높습니다. 웹 내비게이션 태스크라면 전문가가 직접 수백 개의 웹 탐색 과정을 시연해야 합니다.

$$\mathcal{L}_{SFT} = -\sum_{t=1}^{T} \log \pi_\theta(a_t | s_t, a_{1:t-1})$$

이 손실은 시범 궤적 $\{(s_1, a_1), \ldots, (s_T, a_T)\}$ 에 완전히 의존합니다.

### 2. 오류 복구 불가

SFT로 학습된 에이전트는 시범 데이터의 분포(distribution)에서 벗어난 상황에서 취약합니다. **공변량 이동(covariate shift)** 문제입니다. 에이전트가 실수를 하면 분포 외(out-of-distribution) 상태에 진입하고, 이후 행동이 급격히 악화됩니다.

```
시범 분포: s₁ → a₁ → s₂ → a₂ → ...

에이전트가 실수: s₁ → a₁' (잘못된 행동) → s₂' (분포 외)
                                            ↓
                               이후 모든 행동이 부적절
```

### 3. 지식의 상한

SFT 에이전트는 시범 데이터보다 더 나은 전략을 학습할 수 없습니다. 인간 전문가의 한계가 에이전트의 상한이 됩니다. 창의적이거나 새로운 전략의 발견이 불가능합니다.

## RL의 장점

RL은 에이전트 훈련의 이 세 가지 한계를 근본적으로 해결합니다.

### 자율 탐색

RL 에이전트는 스스로 환경을 탐색하며 학습합니다. 시범 데이터 없이도 시행착오를 통해 최적 전략을 발견할 수 있습니다.

$$\pi^* = \arg\max_\pi \mathbb{E}_\pi\left[\sum_{t=0}^{T} \gamma^t r_t\right]$$

### 오류 복구 학습

RL 훈련 과정에서 에이전트는 다양한 오류 상황을 자연스럽게 경험합니다. 이를 통해 오류 후 복구하는 전략을 명시적으로 학습합니다. 이는 SFT의 공변량 이동 문제를 해결합니다.

### 인간 능력 초과 가능

RL 에이전트는 탐색을 통해 인간 전문가도 발견하지 못한 전략을 찾을 수 있습니다. AlphaGo가 인간 바둑 기사들도 몰랐던 수를 발견한 것처럼, LLM 에이전트도 새로운 문제 해결 전략을 발견할 수 있습니다.

### 환경 피드백 직접 활용

환경의 보상 신호가 직접 학습에 활용됩니다. 테스트 케이스 통과, 웹 탐색 성공, 게임 점수 등이 자연스러운 보상이 됩니다.

## 에이전트 RL vs 추론 RL

에이전트 RL과 단순 추론 RL(예: 수학 문제 풀기)은 중요한 차이가 있습니다.

| 특성 | 추론 RL | 에이전트 RL |
|------|--------|---------|
| 에피소드 길이 | 짧음 (수십 토큰) | 긺 (수백~수천 스텝) |
| 환경 상호작용 | 없음 | 필수 |
| 행동 공간 | 텍스트 생성 | 도구 호출, UI 조작 등 |
| 보상 지연 | 즉각 또는 단기 | 장기 지연 |
| 상태 변화 | 내부 추론 상태 | 외부 환경 상태 |
| 비용 | 낮음 | 높음 (환경 실행) |

## 에이전트 RL의 핵심 도전 과제

### 1. 긴 수평선(Long Horizon) 문제

실제 에이전트 태스크는 수십~수백 단계를 요구합니다. "항공편 예약하기"는 웹사이트 열기, 검색, 날짜 선택, 결제 등 수십 개의 행동을 순서대로 수행해야 합니다. 이렇게 긴 에피소드에서 초기 행동에 대한 크레딧 할당이 매우 어렵습니다.

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

시간 차이 오류 $\delta_t$ 는 $\gamma^n$ 항으로 인해 초기 상태에서 거의 0에 가까워진다.

### 2. 희소 보상(Sparse Reward)

많은 에이전트 태스크에서 보상은 에피소드 마지막에만 주어진다. 태스크에 실패하면 수백 개의 행동 중 어느 것이 문제였는지 알 수 없습니다.

**해결 접근법:**
- 중간 보상 엔지니어링 (부분 완료 보상)
- 계층적 RL (서브골 분해)
- 역방향 강화학습 (보상 함수 학습)

### 3. 대규모 행동 공간

LLM 에이전트의 행동 공간은 사실상 무한하다. 텍스트를 자유롭게 생성하거나 임의의 코드를 실행할 수 있습니다. 이는 탐색을 매우 어렵게 만듭니다.

$$|\mathcal{A}| = |\text{vocabulary}|^{L_{max}} \approx 50000^{2048}$$

**해결 접근법:**
- 행동 마스킹 (유효하지 않은 행동 제외)
- 제약된 생성 (문법 기반 행동 공간)
- 계층적 행동 (고수준 계획 + 저수준 실행)

### 4. 환경 실행 비용

각 RL 훈련 스텝마다 실제 환경과 상호작용해야 합니다. 웹 브라우저 시뮬레이션, 코드 실행, 데이터베이스 쿼리는 모두 비용이 발생합니다. 수백만 번의 훈련 스텝이 필요한 RL에서 이 비용은 빠르게 누적됩니다.

**해결 접근법:**
- 환경 모델 학습 (Model-based RL)
- 병렬 환경 실행 (비동기 학습)
- 경험 재사용 (Replay Buffer)

## 정리

LLM 에이전트는 환경과 상호작용하며 목표를 달성하는 자율 시스템으로, SFT 기반 훈련의 시범 의존성, 오류 복구 불가, 인간 능력 상한이라는 세 가지 한계를 갖습니다. RL은 자율 탐색, 오류 복구 학습, 인간 능력 초과의 가능성으로 이 한계를 해결합니다. 그러나 긴 수평선, 희소 보상, 대규모 행동 공간, 환경 실행 비용이라는 에이전트 RL 고유의 도전 과제가 존재하며, 이것이 이후 챕터에서 다루는 각 연구의 출발점입니다.
