## AI 피드백의 등장 배경

RLHF의 핵심 병목은 **인간 피드백의 수집 비용** 입니다. 멀티모달 데이터에서는 이 비용이 더 높습니다. 이미지를 보고 판단해야 하는 작업은 텍스트보다 더 많은 시간과 전문성을 요구합니다.

**RLAIF(Reinforcement Learning from AI Feedback)** 는 인간 대신 강력한 AI 모델이 피드백을 제공하는 패러다임입니다. 텍스트 도메인에서 Claude의 Constitutional AI(2022)로 처음 체계화되었고, 이제 멀티모달 도메인으로 확장되고 있습니다.

그러나 멀티모달 RLAIF에는 특수한 도전이 있습니다. GPT-4V나 Gemini Ultra 같은 폐쇄형 모델에 의존하면 비용, 접근성, 재현성 문제가 생긴다. **RLAIF-V** 는 오픈소스 모델만으로도 충분히 신뢰할 수 있는 AI 피드백을 생성할 수 있음을 보여줍니다.

## RLAIF-V: 오픈소스 AI 피드백

**RLAIF-V** (Yu et al., CVPR 2025)는 오픈소스 멀티모달 모델을 피드백 제공자로 활용하여 GPT-4V를 초과하는 신뢰성을 달성한 시스템입니다.

### 핵심 설계 원칙

**분할 정복(divide and conquer) 피드백**: 이미지-텍스트 쌍 전체를 한 번에 평가하는 것이 아니라, 여러 세부 질문으로 분해하여 각각 평가합니다.

```
전체 응답 평가 → 분해
    ├── "이미지에 언급된 객체가 실제로 존재하는가?"
    ├── "객체의 속성(색상, 크기)이 정확한가?"
    ├── "공간 관계 설명이 올바른가?"
    └── "문장의 논리적 일관성이 있는가?"
```

각 서브 질문에 대한 답변을 집계하여 최종 피드백을 생성합니다. 이 방식은 오픈소스 모델도 전체적인 판단보다 세부적인 판단에서 더 정확하다는 관찰에 기반합니다.

### 오픈소스 AI 피드백 생성 파이프라인

```
1. 후보 응답 생성: π_θ로 N개 응답 샘플링
2. 서브 질문 생성: 이미지 내용 기반 자동 생성
3. 오픈소스 VLM으로 서브 질문 답변
4. 답변 집계 → 선호 레이블 생성
5. 선호 데이터로 DPO 훈련
```

### 성능 결과

| 벤치마크 | GPT-4V | LLaVA-1.6-34B | RLAIF-V (오픈소스) |
|---------|--------|--------------|-----------------|
| MMHal-Bench | 3.5 | 3.1 | **3.8** |
| Object HalBench | 86.4% | 79.2% | **91.3%** |
| POPE F1 | 87.1% | 85.4% | **92.7%** |
| LLaVA-Bench | 93.1% | 88.7% | **96.2%** |

오픈소스 AI 피드백만으로 GPT-4V를 초과한다는 결과는, 폐쇄형 모델 의존 없이도 고품질 멀티모달 정렬이 가능함을 입증합니다.

## LLaVA-Critic: 최초 오픈소스 LMM 범용 평가자

**LLaVA-Critic** (Xiong et al., CVPR 2025)는 멀티모달 모델 평가를 위한 최초의 오픈소스 범용 평가자다. 기존에는 멀티모달 평가를 위해 항상 GPT-4V나 Gemini Ultra 같은 독점 모델에 의존해야 했다.

### LMM-as-a-Judge 패러다임

LLaVA-Critic의 핵심은 **LMM-as-a-Judge** 패러다임입니다. 멀티모달 모델이 다른 멀티모달 모델의 출력을 평가하는 역할을 수행합니다.

**Pointwise 평가**: 단일 응답의 절대적 품질을 1~10 척도로 평가합니다.

$$\text{score}(o, I, q) = \text{LLaVA-Critic}(o, I, q) \in [1, 10]$$

**Pairwise 평가**: 두 응답을 비교하여 어느 것이 더 나은지 판단합니다.

$$\text{winner} = \text{LLaVA-Critic}(o_1, o_2, I, q) \in \{o_1, o_2, \text{tie}\}$$

### 훈련 방법

LLaVA-Critic은 고품질 평가 데이터로 훈련됩니다. 평가 데이터는 두 소스에서 수집됩니다.

1. **GPT-4V 생성 평가**: 다양한 멀티모달 태스크에 대한 GPT-4V의 평가와 이유
2. **인간 선호 데이터**: 기존 멀티모달 RLHF 데이터셋 재활용

이 데이터로 LLaVA 기반 모델을 미세조정하여 평가 전문 모델을 만듭니다.

### 7개 벤치마크 성능

| 벤치마크 | GPT-4o | Gemini-1.5-Pro | LLaVA-Critic-72B |
|---------|--------|----------------|-----------------|
| LLaVA-Bench (Pairwise) | 89.3% | 85.7% | **91.4%** |
| MMVet | 71.2% | 68.9% | **73.8%** |
| MathVista | 63.8% | 65.1% | **67.2%** |
| MMMU | 69.1% | 67.3% | **70.4%** |
| MMHal-Bench | 3.8 | 3.7 | **4.0** |
| WildVision | 72.4% | 70.8% | **74.1%** |
| VQAv2 | 77.8% | 79.2% | **80.3%** |

오픈소스 LLaVA-Critic-72B가 7개 벤치마크 모두에서 GPT-4o와 Gemini-1.5-Pro를 초과했다.

## LLaVA-Critic-R1: 비평이 곧 생성

**LLaVA-Critic-R1** (arXiv:2509.00676)은 흥미로운 관찰에서 출발합니다. **강력한 비평 모델은 강력한 생성 모델이기도 하다** 는 것입니다.

### 비평과 생성의 연결

비평 능력(무엇이 좋은 응답인지 판단)과 생성 능력(좋은 응답을 만들어내는 것)은 근본적으로 같은 이해를 공유합니다. 따라서 비평 모델을 위한 RL 훈련이 생성 능력도 동시에 향상시킨다.

LLaVA-Critic-R1은 이를 명시적으로 활용합니다. 비평 태스크(선호 판단)와 생성 태스크(응답 생성)를 동시에 RL로 훈련합니다.

$$\mathcal{L}_{R1} = \mathcal{L}_{critic} + \lambda \cdot \mathcal{L}_{generation}$$

### Chain-of-Thought 비평

LLaVA-Critic-R1은 점수를 즉시 출력하는 것이 아니라, **추론 과정을 명시적으로 생성** 합니다.

```
[이미지 + 응답 입력]
<think>
이 응답은 이미지의 주요 내용(해변 풍경)을 잘 포착했다.
"파란 하늘과 맑은 바다"는 정확하다. 그러나 "모래사장에
우산이 여러 개"라는 부분은 이미지에서 우산이 두 개만
보이므로 약간 과장되어 있다. 전반적으로는 정확하다.
</think>
점수: 8.2/10
```

이 추론 과정은 더 정확하고 일관된 평가를 가능하게 합니다.

### 선호 판단과 생성 능력의 동시 향상

| 능력 | 훈련 전 | RL 훈련 후 |
|------|---------|----------|
| 선호 판단 정확도 | 72.3% | **89.7%** |
| VQA 정확도 | 74.1% | **82.6%** |
| 환각 비율 | 22.4% | **11.8%** |
| 추론 벤치마크 | 61.2% | **73.4%** |

비평 중심 RL 훈련이 생성 성능도 함께 향상시키는 것은 "평가를 잘하는 모델이 생성도 잘한다"는 직관을 강력하게 지지합니다.

## RLAIF-V vs LLaVA-Critic 비교

| 특성 | RLAIF-V | LLaVA-Critic | LLaVA-Critic-R1 |
|------|--------|-------------|----------------|
| 주요 역할 | 정렬 훈련 | 범용 평가 | 평가 + 생성 |
| 피드백 유형 | 선호 레이블 | 점수 + 이유 | CoT + 점수 |
| 오픈소스 | 완전 공개 | 완전 공개 | 완전 공개 |
| GPT 의존성 | 없음 | 훈련 데이터만 | 없음 |
| 추론 능력 | 낮음 | 중간 | 높음 |
| 생성 능력 | 보통 | 보통 | 높음 |

## 정리

RLAIF-V와 LLaVA-Critic은 오픈소스 생태계에서 멀티모달 AI 피드백과 평가의 새로운 기준을 세웠다. 폐쇄형 모델에 의존하지 않고도 GPT-4V를 초과하는 신뢰성과 평가 능력을 달성한 이 성과는, 멀티모달 AI 연구의 접근성을 크게 높입니다. LLaVA-Critic-R1이 보여주는 "비평이 곧 생성"의 원리는 향후 통합적 멀티모달 에이전트 설계에 중요한 통찰을 제공합니다.
