## 커리큘럼 학습의 동기

인간 교육에서 커리큘럼은 당연한 원칙이다. 미적분을 배우기 전에 대수를 먼저 배우고, 복잡한 소설을 읽기 전에 짧은 글부터 시작한다. 강화학습에서도 동일한 원칙이 적용된다.

RLVR에서 무작위로 샘플링한 문제로 학습하면 두 가지 문제가 발생한다. 첫째, **너무 쉬운 문제**: 모든 시도에서 정답을 맞히면 보상 분산이 0이 되어 학습 신호가 없다. 둘째, **너무 어려운 문제**: 모든 시도에서 오답이면 보상 분산이 없거나 너무 희소하여 학습이 느리다.

최적의 학습은 **현재 모델이 일부는 맞히고 일부는 틀리는 난이도** 의 문제에서 이루어진다. 이 "적정 난이도 구간(Goldilocks zone)"을 체계적으로 관리하는 것이 커리큘럼 RL의 목표다.

## Easy-to-Hard 전략

가장 단순한 커리큘럼은 **Easy-to-Hard** 다. 훈련 초기에는 쉬운 문제로 시작하여 기본 패턴을 학습하고, 점진적으로 어려운 문제로 이동한다.

구현 방법:
1. 전체 문제집을 난이도별로 분류 (정답률, 풀이 단계 수 등으로 추정)
2. 초기 $T_1$ 스텝: 하위 30% 난이도 문제만 사용
3. 중간 $T_2$ 스텝: 하위 60% 난이도 문제 사용
4. 이후: 전체 문제 사용

이 방식의 장점은 구현이 단순하고 재현 가능하다는 것이다. 단점은 난이도 구간 전환 시점을 수동으로 결정해야 하며, 모델 학습 속도에 관계없이 스케줄이 고정된다는 것이다.

## 자기 진화 커리큘럼 (MAB 기반)

**자기 진화 커리큘럼(Self-Evolving Curriculum)** (arXiv:2505.14970)은 **Multi-Armed Bandit(MAB)** 알고리즘을 활용하여 난이도 조절을 자동화한다.

### MAB 프레임워크

각 난이도 구간 $k$를 하나의 "팔(arm)"로 모델링한다. 각 배치에서 어느 난이도 구간에서 문제를 샘플링할지 MAB가 결정하고, 학습 신호(보상 분산, 성능 향상)를 피드백으로 받아 다음 선택을 조정한다.

사용하는 MAB 알고리즘은 일반적으로 **UCB(Upper Confidence Bound)** 또는 **Thompson Sampling** 이다.

$$
k^* = \argmax_k \left[ \mu_k + c\sqrt{\frac{\ln t}{n_k}} \right]
$$

여기서 $\mu_k$는 난이도 구간 $k$의 평균 학습 신호, $n_k$는 방문 횟수, $t$는 전체 스텝, $c$는 탐색-활용 균형 계수다.

### 학습 신호 정의

MAB의 보상은 다음 중 하나로 정의한다.

- **보상 분산**: $\text{Var}(r_1, \ldots, r_G)$. 분산이 높을수록 학습이 활발하게 이루어짐
- **성능 향상률**: 연속된 배치 간 정답률 변화
- **엔트로피 감소**: 정책 엔트로피의 감소율

arXiv:2505.14970에서 MAB 기반 자기 진화 커리큘럼은 고정 커리큘럼 대비 MATH에서 3~5% 향상을 보고했다.

## WebRL의 자기 진화 커리큘럼

**WebRL** (Qi et al., 2024)은 웹 에이전트 훈련에 자기 진화 커리큘럼을 적용했다. 웹 태스크는 난이도가 매우 다양하고(간단한 버튼 클릭부터 복잡한 다단계 탐색까지), 자동 검증(태스크 완료 여부)이 가능하다.

WebRL의 커리큘럼은 **온라인** 으로 동작한다. 모델이 현재 배치에서 생성한 실패 사례를 분석하여 다음 배치의 태스크 구성을 조정한다. 성공률이 너무 높으면 더 어려운 태스크로, 너무 낮으면 더 쉬운 태스크로 이동한다.

## VL-Cogito: 시각-언어 모델용 점진적 커리큘럼

**VL-Cogito** (arXiv:2507.22607)는 시각-언어(Vision-Language) 모델의 다모달 추론 강화를 위한 점진적 커리큘럼을 제안한다.

VL 모델의 추론 난이도는 언어 추론 난이도 외에 시각 정보의 복잡도(해상도, 물체 수, 공간 관계 등)가 추가된다. VL-Cogito는 이 두 차원을 동시에 고려하는 **2D 커리큘럼** 을 설계한다.

| 단계 | 시각 복잡도 | 언어 추론 복잡도 |
|------|----------|--------------|
| 1단계 | 낮음 (단순 이미지) | 낮음 (단순 QA) |
| 2단계 | 중간 | 낮음 또는 중간 |
| 3단계 | 높음 | 높음 (복잡 추론) |

이 점진적 접근으로 MathVista, MMStar 등 시각 추론 벤치마크에서 기존 방법 대비 유의미한 향상을 보고했다.

## Prompt Curriculum Learning

**Prompt Curriculum** 은 동일한 문제를 더 어렵게 만드는 프롬프트를 점진적으로 적용하는 방법이다. 예를 들어, 초기에는 힌트가 있는 프롬프트를 사용하고, 점차 힌트를 제거한다.

$$
x_t = \text{template}(x_{\text{base}}, \text{hint\_level}(t))
$$

이 방법은 새 데이터 없이 기존 문제집을 재활용할 수 있어 데이터 효율적이다.

## 고정 vs 적응적 커리큘럼 비교

| 기준 | 고정 커리큘럼 | 적응적 커리큘럼 |
|------|------------|--------------|
| 구현 복잡도 | 낮음 | 높음 |
| 재현성 | 높음 | 낮음 (확률적) |
| 모델 학습 속도 반응 | 없음 | 있음 |
| 하이퍼파라미터 수 | 적음 | 많음 |
| 최종 성능 | 기준 | 일반적으로 더 높음 |
| 디버깅 용이성 | 쉬움 | 어려움 |

실용적으로는 고정 Easy-to-Hard로 시작하여 기준선을 확립하고, 이후 적응적 커리큘럼으로 성능을 개선하는 순서가 권장된다. 적응적 커리큘럼의 MAB 구성 요소는 학습 다이나믹스를 모니터링하는 진단 도구로도 유용하게 활용할 수 있다.
