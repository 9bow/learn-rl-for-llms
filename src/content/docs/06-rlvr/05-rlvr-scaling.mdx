## 도메인 전이의 핵심 질문

수학과 코딩에서 RLVR로 훈련된 모델은 이 도메인에서만 잘하게 될까, 아니면 훈련 도메인을 넘어 일반적 추론 능력이 향상될까? 이 질문은 RLVR의 실용적 가치를 결정하는 핵심이다.

DeepSeek-R1과 후속 연구들의 실험 결과는 놀라운 사실을 보여준다. 수학·코딩 RLVR로 훈련된 모델이 **훈련에 포함되지 않은 도메인(사실 QA, 의료 추론, 법률 추론 등)에서도 성능이 향상** 되는 경향을 보인다.

## Emergent Reasoning 현상

**Emergent Reasoning** 은 RL 훈련 중 예상치 못한 추론 행동이 자발적으로 출현하는 현상이다. DeepSeek-R1-Zero에서 관찰된 self-reflection, verification, dynamic adaptation이 대표적 사례다.

이 현상의 이론적 설명은 아직 완전하지 않지만, 두 가지 가설이 유력하다.

**범용 추론 전략 가설**: 수학 문제를 체계적으로 푸는 데 필요한 기술들(분해, 검증, 역추론, 케이스 분석)이 실제로 도메인 무관한 범용 전략이다. 수학 RLVR이 이 전략들을 강화하면 다른 추론 태스크에도 적용된다.

**표현 개선 가설**: RLVR이 단순히 추론 행동을 학습하는 것이 아니라, 모델 내부의 표현(representation)을 개선하여 더 풍부한 중간 계산이 가능해진다.

## 도메인 간 일반화 증거

| 훈련 도메인 | 평가 도메인 | 관찰된 효과 |
|-----------|-----------|-----------|
| 수학 | 논리 추론 | 일관된 향상 |
| 수학 | 사실 QA | 소폭 향상 |
| 코딩 | 수학 | 향상 (공유 구조) |
| 수학+코딩 | 일반 언어 이해 | 혼재 (태스크별 상이) |

일반화 효과는 태스크 간 구조적 유사성에 비례하는 경향이 있다. 수학과 코딩은 둘 다 체계적 절차적 사고를 요구하므로 강한 전이를 보이고, 창의적 글쓰기 같은 비절차적 태스크로는 전이가 약하다.

## RLVE: 적응적 검증 환경

**RLVE(Reinforcement Learning with Verifiable Environments)** (arXiv:2511.07317)는 검증 환경 자체를 적응적으로 관리하는 프레임워크다. 단일 고정 검증기 대신, 태스크 난이도와 모델 학습 상태에 따라 검증 환경을 동적으로 조정한다.

핵심 아이디어는 **과제 생성기(task generator)와 검증기(verifier)의 공동 진화** 다. 모델이 현재 검증 환경에서 충분히 잘하면, 더 어려운 검증 환경으로 이동한다. 이는 커리큘럼 학습의 자동화된 형태다.

## Knowledge-to-Verification: 지식 집약 도메인 확장

수학과 코딩을 넘어 의료, 법률, 과학 등 **지식 집약 도메인** 으로 RLVR을 확장하려는 시도가 있다.

이 도메인들은 완전한 자동 검증이 어렵다. 그러나 부분적 검증은 가능하다.

- **의료**: 진단 기준(ICD 코드, 진료 가이드라인)과의 일치 여부
- **법률**: 판례 인용의 정확성, 조문 번호의 정확성
- **과학**: 수치 계산의 정확성, 단위 일관성

이러한 부분 검증을 통합하면 완전한 RLVR은 아니지만, 검증 가능한 측면에서 학습 신호를 얻을 수 있다.

## RLVR의 스케일링 특성

RLVR은 두 가지 축에서 스케일링된다.

**모델 크기 스케일링**: 더 큰 모델은 동일한 RLVR 훈련에서 더 큰 성능 향상을 보인다. 이는 큰 모델이 이미 더 풍부한 추론 능력을 잠재적으로 가지고 있어 RL이 이를 더 효과적으로 이끌어낼 수 있기 때문으로 해석된다.

**데이터 스케일링**: 검증 가능한 문제의 수와 다양성을 늘리면 성능이 향상된다. 단순히 같은 문제를 반복하는 것보다 새로운 유형의 문제를 추가하는 것이 더 효과적이다.

## RLVR의 한계와 미래

**현재 한계**:
- 검증 가능 도메인으로 적용 범위가 제한됨
- 이진 보상의 희소성으로 인한 학습 어려움 (쉬운 문제와 어려운 문제 사이의 학습 신호 불균형)
- 검증기 오류(False Positive, False Negative)가 학습을 오도할 수 있음
- 검증 불가능 도메인으로의 전이가 완전하지 않음

**미래 방향**:
- 검증기 자체를 학습으로 구축하는 반자동화 RLVR
- 다단계 검증(과정 + 결과)의 결합
- 검증 불가능 도메인에서의 AI 피드백(RLAIF)과 RLVR의 하이브리드
- 더 넓은 도메인으로의 검증 가능 태스크 확장

RLVR은 LLM 훈련의 패러다임을 바꾸고 있으며, 검증 가능한 도메인에서의 성공이 더 넓은 AI 능력 향상으로 이어지는 경로를 열어가고 있다.
