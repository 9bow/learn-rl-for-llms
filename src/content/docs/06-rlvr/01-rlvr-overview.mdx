## RLVR의 정의

**RLVR(Reinforcement Learning from Verifiable Rewards)** 는 RL에서 사용하는 보상 신호를 학습된 보상 모델이 아닌 **외부 검증기(verifier)** 로부터 얻는 패러다임이다.

검증 가능한 보상의 핵심 요건은 두 가지다.

1. **결정론적 판별**: 응답의 정오(正誤)를 확률이 아닌 이진값으로 판별할 수 있어야 한다.
2. **자동화 가능성**: 인간 개입 없이 프로그래밍으로 검증이 가능해야 한다.

이 요건을 만족하는 대표적 도메인은 다음과 같다.

| 도메인 | 검증 방법 | 보상 신호 |
|--------|---------|---------|
| 수학 문제 | 정답 비교 (수치/기호) | 정답=1, 오답=0 |
| 코딩 | 단위 테스트 실행 | 통과=1, 실패=0 |
| 형식 증명 | 증명 검증기 (Lean, Isabelle) | 검증됨=1, 실패=0 |
| 퍼즐/게임 | 규칙 기반 판별 | 승리=1, 패배=0 |

## 보상 해킹의 원천 차단

RLHF에서 가장 심각한 문제 중 하나는 **보상 해킹(reward hacking)** 이다. 모델이 실제로 좋은 응답을 생성하는 대신, 보상 모델이 높은 점수를 주도록 편향된 응답을 생성하는 현상이다. 예컨대 보상 모델이 긴 응답을 선호하면 모델은 불필요하게 장황한 응답을 생성하고, 특정 단어 패턴을 선호하면 그 패턴을 남용한다.

RLVR은 이 문제를 구조적으로 차단한다. 검증기는 학습된 모델이 아니라 **수학적/논리적 규칙** 이므로, 모델이 검증기를 속이려면 실제로 올바른 답을 생성해야 한다. 단위 테스트를 통과하려면 실제로 작동하는 코드를 작성해야 하고, 수학 검증기를 통과하려면 실제로 정답을 계산해야 한다.

$$
r(x, y) = \mathbb{1}[\text{verify}(x, y) = \text{correct}]
$$

이 이진 보상은 단순하지만 해킹이 불가능하다.

## RLHF 대비 단순성

RLHF는 다음 구성 요소를 필요로 한다.

1. 인간 선호 데이터 수집 (비용 높음)
2. 보상 모델 학습 (추가 모델, 오류 전파 가능)
3. 보상 모델 기반 RL (보상 해킹 위험)
4. 지속적인 보상 모델 갱신 (분포 이동 대응)

RLVR의 구성 요소는 훨씬 단순하다.

1. 검증 가능한 문제-정답 데이터셋 (수학 문제집, 코딩 챌린지 등 기존 자원 활용)
2. 검증기 (수식 계산기, 단위 테스트 실행기 등 기존 도구 활용)
3. RL 알고리즘 (GRPO, PPO 등)

보상 모델 학습 단계가 완전히 제거되므로 오류 전파 경로가 없고, 파이프라인이 단순하며, 재현성이 높다.

## RLVR의 확장 가능성

RLVR이 RLHF보다 확장 가능한 이유는 **데이터 획득 비용** 에 있다.

RLHF는 인간 선호 주석이 필수이므로 데이터 규모를 늘리기 어렵다. 반면 RLVR은 기존의 수학 문제집, 프로그래밍 챌린지(LeetCode, Codeforces), 형식 수학 라이브러리(Mathlib) 등을 그대로 사용할 수 있다. 이러한 자원은 이미 수십만~수백만 개 규모로 존재한다.

또한 RLVR에서는 새로운 도메인에 진입할 때 필요한 것이 해당 도메인의 검증기뿐이다. 검증기를 구축하면 즉시 RL 학습이 가능하며, 인간 주석 수집에 수개월이 소요되는 RLHF와 달리 빠르게 확장할 수 있다.

## RLVR의 한계

RLVR은 강력하지만 적용 범위가 제한된다. 검증 가능한 객관적 정답이 없는 도메인(창의적 글쓰기, 오픈엔드 대화, 복잡한 윤리적 판단 등)에서는 적용이 어렵다. 이 경우 RLHF나 AI 피드백(RLAIF) 등의 보완적 방법이 필요하다.

현재 연구의 방향은 RLVR을 검증 가능 도메인에서 먼저 강력한 추론 능력을 키운 뒤, 이를 검증 불가능 도메인으로 전이(transfer)하는 것이다. DeepSeek-R1이 이 접근의 대표적 성공 사례다.
